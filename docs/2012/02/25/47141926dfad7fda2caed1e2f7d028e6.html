 SAP 采购订单显示含税价制作方案

 

轻松解决SAP系统采购信息计量中物料价格不能保存含税价问题


我们在和供应商谈价时，大部分国内供应商的报价都是含税的，然而我们现在在系统中维护采购信息记录时，

只能输入不含税价，所以在创建采购订单时，将含税价转成不含税价后，小数点后很多位，造成价格不准。

影响财务发票校验，影响套打打印格式美观。


经过与实施方交流，最终形成四个解决方案：


1. 手工维护采购信息记录时，使用计算器，算出净价，并将净价放大1000倍，能解决大约90%的问题；
2. 手工维护采购信息记录时，可以利用“价格单位”，例如，税率是17%，

    就在“价格单位”上输入117（应该是1.17，由于不能输入小数，所以输入117），在“净价”处输入含税价，

    含税价扩大100倍，例如本来某物料的含税价是950元/吨，现在输入95000，这个就是利用净价字段来输入

    含税价！  系统计算时，95000/117，得出的是净价。
3. 在采购订单中配置条件类型，在采购信息记录中的“净价”出输入含税价，“价格单位”为1，

    在采购订单中转成含税价时使用除法，算出含税价和净价；
4.  在采购信息记录中配置条件类型，在“净价”出输入含税价，“价格单位”为1，增加“进项税”，

   保存时采购信息记录时，转成含税价。再在采购订单中使用乘法，算出物料的含税价和净价。

   由于项目已经上线，虽然第3种和第4种方案是最佳的解决方案，但考虑到工作量和关乎项目的成败，

   虽在测试系统中测试成功，仍未采用。


比价第一种和第二种方案，其中第2种方案最为便利，对系统中已存在的信息记录，待日后再修改。

最后确定采用第二种方案！
