集成Fabric的Crashlytics(崩溃统计)

  • 柚子社区

最近使用Google Play的过程中发现console改版后的崩溃统计功能似乎很不好用了，所有就有了这个文章，Fabric在2017年初已经被Google从twitter手中收购了，所以说集成fabric也算是google自家产品了。
一、懒人模式
直接看官方的向导吧：https://fabric.io/downloads/android-studio
简单几步即可集成。
二、手动模式
1.在工程目录的build.gradle中添加fabric maven库及fabric-gradle插件

buildscript { repositories { maven { url 'https://maven.fabric.io/public' }
  }

  dependencies { classpath 'io.fabric.tools:gradle:1.24.1' }
}

2.在app下的build.gradle目录中添加如下内容

apply plugin: 'io.fabric'
repositories { maven { url 'https://maven.fabric.io/public' }
}

3.添加库依赖

  compile('com.crashlytics.sdk.android:crashlytics:2.6.8@aar') {
    transitive = true;
  }

4.配置key在AndroidManifest.xml文件中

      <meta-data
          android:name="io.fabric.ApiKey"
          android:value="xxxxxxxxxxx"
      />

ok，这样就配置好了。
这里有个地方跟其它的sdk不太一样，就是添加APP的方式。
当你登录fabric.io后点击右上角的setting按钮，然后出现如下界面，点击apps

然后点击add按钮

但此时会弹出然你选择添加什么功能。选择第一个就是Crashlytics

此时就会进入配置向导页面，此时页面中的配置内容已经是你新APP的key了

但是此时app并没有添加成功，只有你真正配置成功，并运行APP后，fabric的后台才会添加这个个新的APP。
第一次用这种流程配置SDK，有点不习惯，但是用过之后感觉体验很不错。
1.不会出现随意添加app的混乱。
2.不用自己找apikey，向导的配置文件中就是你的新key。
That’s all. Enjoy it.
