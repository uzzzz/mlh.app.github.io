docker中使用gitlab笔记

  • 柚子社区

.最近在学微服务，想搭建一个gitlab自己玩玩，过程中遇到了很多问题，最后总算是弄好了，主要原因是对docker不熟，走了很多弯路，但感觉还是有收获的。

1.在虚拟机cent OS 7上安装docker

执行以下命令 ，一路输入y，等待安装完
 yum install docker

2. 启动 docker 服务， 并将其设置为开机启动：
  1. [root@localhost ~]# service docker start
  2. [root@localhost ~]# chkconfig docker on
3.检查是否安装成功
输入 docker version 




5. docker images 查看已下载的镜像
docker ps -a 查看docker里的容器

6.可能下载的时候很慢，可以使用阿里云加速器 ，速度回提高很多
https://cr.console.aliyun.com/

2.下载docker compose

使用命令docker search gitlab会查出很多gitlab镜像

比如 GitLab 官方提供的  gitlab/gitlab-ce ，该镜像包含了运行 GitLab 所需的所有内容，只需要这一个镜像就能使用 GitLab。而由 sameersbn 提供的  sameersbn/gitlab  则将 GitLab 分成了三个不同的镜像：数据库、日志、GitLab 服务，这样做的好处便于解耦，以及符合 Docker 的设计原理：轻量级、单一的镜像功能。在此，我们使用  sameersbn/gitlab ，并结合提供的 Docker Compose 文件来启动 GitLab。

参考官网 https://docs.docker.com/compose/install/
执行以下命令 
sudo curl -L https://github.com/docker/compose/releases/download/1.16.1/docker-compose-`uname -s`-`uname -m` -o /usr/local/bin/docker-compose
sudo chmod +x /usr/local/bin/docker-compose
docker-compose --version

下载 docker-compose.yml 文件下载到本机，

wget  https://raw.githubusercontent.com/sameersbn/docker-gitlab/master/docker-compose.yml

3.启动gitlab

docker-compose up


docker-compose会帮你下载所需要的镜像，并启动gitlab



下载完成后，在浏览器输入虚拟机IP地址+端口10080，出现gitlab页面则启动成功
修改密码登入后
