.net core 1.x 2.0独立部署和框架依赖部署

  • 柚子社区

.net core 1.x的部署方式在官网有很详细的说明https://docs.microsoft.com/zh-cn/dotnet/core/deploying/deploy-with-vs

还是稍微总结一下，因为使用VS较多，所以以VS的部署方式为例

有无第三方依赖项就不做区别说明，反正使用NuGet安装之后自动会修改csproj文件


.net core 框架依赖部署： 在这点上，1.x 和 2.0并没有区别，正常的生成，发布即可




独立部署：区别主要在于对于csproj文件的修改

首先，都需要添加RuntimeIdentifiers标签

对于1.x, 填写的内容  win10-x64;osx.10.11-x64

详细的系统支持列表https://docs.microsoft.com/zh-cn/dotnet/core/rid-catalog

完整的csproj文件内容如下

<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>Exe</OutputType>
    <TargetFramework>netcoreapp1.1</TargetFramework>
    <RuntimeIdentifiers>win10-x64;osx.10.11-x64</RuntimeIdentifiers>
  </PropertyGroup>
  <ItemGroup>
    <PackageReference Include="Newtonsoft.Json" Version="10.0.3" />
  </ItemGroup>
</Project>

对于2.0，填写的内容稍有区别，win-x64;linux-x64;osx-x64

完整csproj文件内容如下

<Project Sdk="Microsoft.NET.Sdk">

  <PropertyGroup>
    <RuntimeIdentifiers>win-x64;linux-x64;osx-x64</RuntimeIdentifiers>
    <OutputType>Exe</OutputType>
    <TargetFramework>netcoreapp2.0</TargetFramework>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="Newtonsoft.Json" Version="10.0.3" />
  </ItemGroup>

</Project>

修改之后重新生成项目

再选择目标平台点击发布



然后在将整个输出目录拷贝到目标平台主机上运行即可。
