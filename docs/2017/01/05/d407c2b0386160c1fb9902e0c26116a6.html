    从上面我们知道音频所能够占用的flash空间如下：

#define AUDIO_FLASH_START_SECTOR        0x100
#define AUDIO_FLASH_STOP_SECTOR        0x1AE
#define AUDIO_FLASH_LEN                0xAE000
#define AUDIO_FLASH_READ_START_ADDR    AUDIO_FLASH_START_SECTOR * SPI_FLASH_SEC_SIZE + WAV_HEAD_LEN

#define SPI_FLASH_SEC_SIZE  4096    /**< SPI Flash sector size */

   也就是说音频的flash扇区从0x100到0x1AE,一共0xAE（174）个扇区，而每个扇区的大小为4096个字节，即音频所占用的flash为：174*4096=712704Byte=712.704kByte

        我们写flash的整体思路是：

       1.创建一个扇区大小的数组：static char __attribute((aligned (4))) buf_temp[SPI_FLASH_SEC_SIZE];，注意：这个数组必须是四个字节对齐，因为flash的写操作要求源和目的的地址必须是四个字节对齐

        2.当flash写函数被调用的时候首先判断当前要写的数据大小是否大于一个扇区，如果不大于，那么就把要写进flash的数据先储存到buf_temp数组，如果大于那就进行flash的写操作。

        3.因为flash的擦除操作是整个扇区擦除，所以我们这里对于flash的写操作也是整个扇区的写，当发现传入的数据加上之前储存在buf_temp数组里的数据大于一个扇区的时候这里将会先把buf_temp数组和传入的数据的一部分凑齐一个扇区然后写进flash。

         4.当写入了buf_temp数组和传入的数据的一部分凑齐的一个扇区的数据后，将判断剩下传入的数据是不是扇区大小的整数倍，如果剩余要写的数据时扇区大小的n个整数倍加上m个剩余的字节，这里首先要写的是n个扇区。

         5.然后这里就剩下了m个剩余字节的数据，这个剩余字节的数据又分别两种情况，一种是目前已经写进flash的数据大小加上m大于或者等于整个WAV的音频数据，这种情况下就把m个字节的数据存进flah中，然后结束wav的写入flash的操作。另外一种情况是已经写进flash的数据大小加上m小于整个wav音频文件的数据大小，子时候把m个字节的剩余数据拷贝到buf_temp数组中。、

         6.如果wav的数据写入操作还没有完成，这里将继续等待下一个要存储的数据包的到来。退出 write_flash_music函数。
