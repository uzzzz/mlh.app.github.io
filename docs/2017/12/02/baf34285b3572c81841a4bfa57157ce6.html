Shiro使@RequestMapping失效，调用404异常

  • 柚子社区
版权声明：本文为博主原创文章，未经博主允许不得转载。 https://blog.csdn.net/qq_35981283/article/details/78698309

使用springboot集成shiro的时候总会有很多坑，现在又遇到一个。

shiro集成进来后，调用API直接404异常。
部分代码如下：

    @GetMapping("/user")
    @RequiresPermissions(value={"user:add","resource:delete"},logical = Logical.OR)
    public  User getUserInfo(@RequestParam(value = "crsKey") String username){
        return userService.findByUsername(username);
    }

如果把RequiresPermissions这行去掉，是可以正常访问的，加上之后就是404。
被这个问题困扰了两天，现在直接给出解决方案，ShiroConfig中的代码：

    @Bean
    public static DefaultAdvisorAutoProxyCreator getDefaultAdvisorAutoProxyCreator(){

        DefaultAdvisorAutoProxyCreator defaultAdvisorAutoProxyCreator=new DefaultAdvisorAutoProxyCreator();
        defaultAdvisorAutoProxyCreator.setUsePrefix(true);

        return defaultAdvisorAutoProxyCreator;
    }

在shiro的配置中加入defaultAdvisorAutoProxyCreator.setUsePrefix(true);这个设置就OK了。
主要是我跟其它项目集成的时候遇到这个问题，单个shiro项目没有出现这种情况。
我的shiro代码可以参考前面的文章Spring Boot集成无状态Shiro–内容详细介绍。

我这里是用springboot解决的，springmvc可其它框架的可以参考以下资料：
http://www.jianshu.com/p/b27387b97292
http://www.iteye.com/problems/94650

稍微多说两句，@RequiresPermissions这类shiro权限的注解，是DefaultAdvisorAutoProxyCreator这个bean设置之后才会生效的，所以在网上搜索资料后我想到了在这个地方设置true。相对来说针对springboot的参考资料还是太少了，网上基本都是针对spring mvc的使用xml配置的解决方案，很多时候知道了解决方案却不知道怎么在springboot中设置，希望springboot快速的强大起来（可参考资料越来越多）。
