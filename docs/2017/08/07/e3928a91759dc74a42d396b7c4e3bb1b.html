面对区块链行业的发展困境，他们想啃“技术”上的硬骨头

本堂茜

渡鸦区块链专栏记者



内容预读


区块链技术经过几年的发展,逐步展现出其潜力,开始在一些 领域落地。但是,作为一项新兴技术,仍存在诸多技术瓶颈。其中, 易用性和 TPS(Transactions per Second)是制约目前区块链应用落 地的重要原因。易用性制约了企业的开发进度,导致仍然没有一款 杀手级的区块链应用出现。而对于需要高并发的业务,目前也没有 区块链技术解决方案来满足。

社区的一些优秀代表都在积极推动区块链技术的发展,在各自 领域深耕细作,取得了长足的进步。但受限于当前技术的发展状况,TPS 成为各个平台都面临的难题,TPS 3000 成为行业的共同的瓶颈, 使得区块链在高价值的高并发业务领域无法落地。

综上所述,行业急需一个支撑 BAT 用户级别的海量高并发运用 场景的区块链底层平台。HPB 应运而生,旨在解决这个行业瓶颈。HPB 将提供高频率访问需求的智能合约业务。它除了能够实现中心 化服务器的用户体验,还会支持中心化服务器无法承载的千亿级终 端的超大规模物联网场景。 

‍

‍


技术出身的汪晓明，最早在某跨境电商平台担任CTO，主要负责跨境出口贸易。后来他加入到银联做大数据相关业务。不断涉及到的跨境出口贸易，让他们在跨境支付方面遇到了难题：最常用的Paypal，不仅交易费昂贵，也存在着诸多问题。


在寻找新的支付方式的过程中，他关注到比特币。这个技术和金融体系深深地吸引了他，也让他接触到了许多和他志同道合的区块链爱好者，后来汪晓明决定利用自身的技术优势，在这个很有前景的行业内创业，组建了这只区块链团队，成立了朝夕网络，做区块链技术支持的金融解决方案服务。

 

HBP创始人 汪晓明


然而在前期区块链创业中，各种方向、模式都是迷茫的。汪晓明自然也遇到了很多困难，在技术人才本来就稀缺的情况下，招人自然很困难，加上早期风投对这个新兴领域看不懂，不敢轻举妄动，在资金方面他们也时常捉襟见肘。“为了创业我们投入很大，可以说是all in了。” 后来随着人们对区块链的了解加深，和项目机会的增多，才逐渐好转。最早他们从比特币切入，但是他们逐渐发现比特币不适合做商业化落地，于是转向更适合落地的以太坊，当然他们对业内主流方案也有深入的比较和研究。


 

‍

‍


然而一年下来他们发现，真正的商业化落地并不简单，需要扎实的技术基础。在现在整个底层技术不够成熟的情况下，会遇到许多技术难题，只有正真的理解行业需求并与行业结合才能够真正的落地。但即使困难重重，他们仍然取得了一些成绩，近期他们给某银行做的区块链积分的项目，这个方案主要是通过搭建联盟链来实现，会邀请多家银行加入，银行是核心节点，也会部署部分合作的大商户，通过联盟链来打通多家银行的积分兑换，让客户的积分更加有效的流转和利用起来。

 

在与行业结合的落地实践中，汪晓明看到了区块链技术本质上的一些不足，区块链作为一项新兴的技术还有很多瓶颈要突破，很多行业的需求在等待着技术的发展。

 

汪晓明说，目前易用性和 TPS(Transactions per Second)是制约目前区块链应用落地的重要原因。易用性制约了企业的开发进度，导致现在仍然没有一款杀手级的区块链应用出现。而对于需要高并发的业务，目前也没有好的区块链技术解决方案来满足。


 

‍

‍


因此，汪晓明发起了HPB（High-performance Blockchain）——“芯链”项目。HPB（芯链）是一种全新的区块链软硬件体系架构，其定位为易用的高性能区块链平台，为每秒提供数百万交易确认，跟产业深度结合从而满足现实世界的真实商业需求。旨在解决目前的技术瓶颈，实现一个可以支撑 BAT 级别的海量高并发运用场景的区块链底层平台。

 

汪晓明介绍说，HPB（芯链）的创新点在于它是一种全新区块链体系架构。其中包含芯片加速引擎和区块链底层平台,旨在实现分布式应用的性能扩展。


HPB通过创建一个可以构建应用程序的类似操作系统的架构来实现。该软件架构提供帐户，身份与授权管理、策略管理、数据库、异步通信以及在数以千计的CPU、FPGA或群集上的程序调度，该区块链每秒可以支持数百万个交易，且达到秒级确认。HPB还设计了专用的区块链硬件服务器，通过结合CPU串行能力和FPGA等定制芯片的并行能力，提升高频运用场景。

 


“HPB是一个社区开源项目，对于大型软件来说，只有让更多的开发者参与进来，才能实现产品的快速迭代。”汪晓明说HPB的设计走向---未来将成为一个支持亿级用户的平台，满足类似BAT这样的高并发交易场景性能需求的企业，每天需要处理数亿的日活用户交易的需求，而且为了保证用户体验，他们将实现秒级的确认时间。


 

‍

‍


对于无法并行的应用场景比如交易所，HPB将提供更强大的串行能力。HPB采用TOE技术，通过专用网卡上专用处理器来完成一些或所有数据包的处理任务。也就是说，通过采用配有 TOE 芯片的专用网卡，包括 TCP 在内的四层处理请示都可以从主机处理器转移到硬件加速卡上，其最终的结果就是在加速网络响应、并发能力增强同时降低服务器复杂度，提高节点处理性能。

 

此外，HPB还想改变节点之间数据串行的处理的方式，通过设计专用的区块链硬件加速单元(包括加速硬件及加 速硬件固件)以及与之匹配的软件引擎（硬件加速系统层驱动以及上层软件接口 API），通过结合 CPU 串行能力和 FPGA、GPU、ASIC 等芯片的并行处理能力，实现高性能和高并发计算加速解决网络层次结构复杂、延迟时间长、串行处理性能低等用户体验的问题。

 


‍

‍


HPB采用DPOS的共识算法，由 DPOS 共识算法维护的区块链出块者都是 100%在线，DPOS 共识算法可以广泛支持公有链、联盟链与私有链的场景，如果有业务场景需要用到满足特定业务需求或目的的共识算法， HPB 的可插拔共识算法模块可以灵活支持不同共识算法的集成切换，并且对DPOS 类的算法提供通用的共识步骤接口。

 

“HPB宣布支持NeoVM（NEO虚拟机，原小蚁虚拟机AVM）。”汪晓明解释到，HPB采用的是通用虚拟机的形式，支持多种虚拟机。同时可以随着时间推移持续按需求增加新的虚拟机。HPB 会实现轻量化的智能合约虚拟机，支持多层次的智能合约 虚拟机体系，底层的虚拟机与上层高级程序语言解析转换相结合，HPB灵活支持虚拟机的基础应用。通过定制化的 API 操作实现虚拟机的外置接口，可以灵活地与账本数据以及外部数据进行交互操作。这一机制实现了智能合约运行时达到原生代码执行的高性能，同时也实现了支持不同区块链的通用虚拟机机制。


 

‍

‍


目前HPB“芯链”的核心团队成员大部分自于华为、英特尔、IBM、中软等国际国内一流的硬件和软件企业。团队早年都关注比特币为代表的数字货币，对各个主流平台都比较了解，同时也比较务实地在推动金融商业化的落地。

 

汪晓明的团队也提供区块链专家顾问服务，提供一站式专业的区块链金融解决方案。汪晓明本身也会在博客上、各大专栏上无偿撰写技术文章、为机构做技术培训等。对区块链行业的发展，他也有不少自己的独特看法和见解，并且是国内第一批以太坊的技术研究者和技术布道者。他认为区块链行业未来5年一定是爆发期，虽然过程中难免会有波动，也许现在还存在着很多投机风险，但是从长远来看，行业一定是看好的，长期投资没问题。

 

汪晓明相信区块链和互联网一样会渗透到各个行业，但是会有先后顺序，从区块链历史来看，首先应当是是数字资产领域，偏数字资产领域更容易落地，未来解决了区块链性能和高频交易瓶颈问题后，会在其他领域包括身份认证、存证、众筹、供应链、物联网、征信等领域进行深度结合和快速的发展。

 


然而现有的底层技术还不够成熟，尽管以太坊经过2年的发展相对强势，fabric在性能方面也不错，但是各自都有不够合理的地方。汪晓明认为应该要有更好的底层出现，才能真正引爆区块链的爆发。他预估这样的底层平台应该未来1年内会出现。

 

在现实所逼下，很多项目找不到合适的链，于是会自己进行底层改造，但效果如何很难评估，由于信息披露的缺乏，很多项目值得商榷，需要进一步的了解才能判断。


 

‍

‍


对于目前几个热门的区块链创业方向，如溯源、版权登记等，汪晓明认为如果做好了价值肯定很大，但是目前还是需要理性评估：溯源怎么做才能让物体和区块链做深度捆绑？如何标识才能保证唯一性？打二维码是可行的方式，但是和区块链的匹配还是成问题的。标识受制于硬件，高价值物品可以绑定芯片，但是成本高昂，不适合单品价值低的领域。钢铁行业可以做打标，但是事实上涉及到切割、加热，在加工过程中标识会变。根据品类不同，需要考虑的地方非常多。

 

今年，汪晓明希望HPB实现多个应用场景和商业模式，包括共享医疗经济、普惠金融智慧大数据、区块链积分系统等。也会投入一部分精力继续研发底层。“我们是抱着开放的态度，把自己的东西共享出来，希望大家能共同维护一个比较通用的底层，再在各自的场景里面做深做精。这样的生态才是最好的，因为各自的能力和经验都有限的，大公司或小公司都没有这种能力能主导一切。”
