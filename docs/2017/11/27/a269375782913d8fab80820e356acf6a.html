区块链，分布式商业的“命中注定”？

区块链并非什么新鲜概念。某种程度而言，它是分布式架构的一个特化。其核心技术本质是通过全冗余或全同步的分布式架构，来实现业务上的可信或协作平台。

有人问，区块链这种通过计算资源和存储资源的冗余，或某种意义上的“浪费”，来达到业务的方便性是否有必要？有，因为它能建立更可信的价值传输通道。

从技术体系来讲，这是技术发展的必由之路；计算和存储资源的成本足够低也使其更具可行性。而从必要性而言，例如怎样在共享经济的所有者和使用者之间进行清算结算和利益分配，对于区块链来说这是个非常合适的使用场景。区块链的技术本质或者说核心定位就是企业间、企业与个人的协作平台，同时也是链接分布式商业的基础设施。

区块链底层是有很多连接的。比如说在携程上购买机票延误险，（在机构层面）就是携程出票，延误险的请求转给保险公司，保险公司确认用户已买机票，再处理买延误险，再返回给携程，携程再返回给个人，这个是传统模式下的正常业务流程。但通过区块链，可将保单数字资产化，以智能合约的模式体现：个人只要把保单的状态设定成Apply（申请中），到携程就变成Pre-approved（待审核），到保险公司就变成Approved（已批准）。在这一模式中，各家机构只需关注自身职责，进入区块链的节点，完成自己的权限即可，单点业务复杂度大大降低。协作成本也会降低。

从商业模式来说，可见淘宝的C2C和京东B2C各有优缺点，而区块链模式可结合二者优点，达到规模大成本低，不需要全都自营而是可以通过协作来完成，但对外可表现为一个整体的形象。

  1. 共享数据控制权。如海淘假货，通过区块链模式，物流公司不可能单方面修改数据，提升可信度。

  2. 最佳系统资源配置。其实银行有大量闲散资金想放出去，但在原有模式下释放成本太高，比如一两千的小额贷款，银行按正常流程走要可能成本就要一千块钱，很难实现。通过区块链处理整合后，单次业务成本可降到几毛钱。

  3. 最终实现轻资产合作运营。分布式商业的数据直接放在区块链上，以防第三方被收买。虽然不能完全杜绝造假，但能大大提高造假成本，信任由此而来。

公链和联盟链是目前区块链应用较多的形式，但对于数字货币而言，二者各有限制性：公链的概率一致性会其分叉，但这在金融级应用是不可能被接受的，性能较低；而联盟链目前大多基于点对点的PBFT算法网络规模同样受限。

在技术特性这一环节，李伟博士介绍了趣链目前所使用的区块链技术，其中共识算法可满足绝大多数中低级业务需求。而业务隔离则较加密、性能等都更为重要：比如供应链过程中有若干家物流企业，要避免非相关方得知其他企业的数据，隐私保护完善才能把业务做起来。

应用场景之一：数字资产

和银行、券商等金融机构的合作，是区块链落地应用中极为重要的场景之一，本节李伟博士运用了大量生动有趣的案例，详细讲解了数字票据、贸易金融、供应链金融、资金流、物流供应链、跨境汇款、ABS（资产证券化）场外交易、债券交易和数据交易这十大业务，以及联盟链的前沿研究。

数字票据和以前的电票有什么区别？比如A欠B一百万，B欠C三十万，那B企业可以把应收账款一百万里的七十万留着，三十万给C，中间不需要填写和质押，可以直接进行拆分，找金融机构贴现和质押的时间和资金成本可以省掉，金融流转效率更高。

供应链金融。细化而言，从供应商到商品出库到物流，每个环节都有对应的订单、仓单和运单编号，整体上环环相扣。区块链可以把供应链的上下游通过信息流的模式绑定在一起，形成一个完整的证据链条，如前所述，单方面作假成本是非常高的，供应链的信息因此可靠。

物流供应链。比如在融资一环，中小物流企业同样具有资金周转需求，但对银行来说，核实中小企业的物流信息确实困难。而在区块链中，运单生成后直接调用外部大数据（国家物流信息平台）来确认这个订单的真实性：平台上有全国所有高速公路的数据，每辆车几时上下高速皆有记录，这就解决了信息流上链的一个可靠性问题。第二点，企业要拿应收账款去做质押去融资，去付第三方的各种费用，但金融机构是无法控制企业拿所付现金去炒股票买房子的，因此可以发起票据或者应收账款发行凭证的申请审批，审批成功之后票据交付第三方再换钱——信息流真实可靠，资金流真实可控，真正把区块链的价值挖掘出来。

ABS。京东百度都在做，这是个由若干机构多方协作，又分步骤的，数字资产生成登记交易转让的过程，区块链的好处在此发挥得淋漓尽致。如果遇到物流链里的金融机构不愿响应融资需求，那么其中的应收账款可以进行合规的包装或者拆分变成ABS——区块链的第一个魅力是由于信任提高效率，第二个是降低整个系统拓展的边际成本，而非系统成本。智能合约就能完成应收账款和ABS的对接，资产进行打包重组拆分的成本降低，整个网络会越来越大。

应用场景之二：数字存证

本节则探讨了电子存证、在线投标保函、政务数据流转平台和股票审查四种场景下，区块链是如何大展身手的。

电子存证。理论上所有发行银联卡的银行都会放一份数字影像的哈希值在区块链上，那么用户可以拿着数字影像去链上查询，来保证自己消费凭证的真实，单是小票成本就可减少120万。类似的合同或契约都可以变成区块链的模式，让数据跑腿，让老百姓少跑腿。区块链在存证领域大有可为，因为合同本身就是应该点对点、去中心的，现在有大量第三方签约平台都转向了区块链。

区块链五大价值，直击应用关键

为什么说区块链的要求不是性能，而是信任先行？如何以区块链优势特性，搭建起独有的信任网络？

  1. 目前区块链遇到的问题是个非单纯技术方案。此前多为科技部对接，现在出现大量的业务部门、资管部，必须要承认技术+业务才是区块链最合适的一个点；

  2. 区块链并非意味着一味的去中心化，必须有些中心化来进行统一的规则设计。比如比特币的私钥一旦丢失，无法补办，账户内的比特币即成为无人认领的死币。但在票据业务上，私钥丢失不可能不补办，必须有一些中心化的设计来进行私钥的管理和补办；

  3. 数据隐私是一个前提条件，通过技术加密保证这一项才能把业务做起来：既要有合作又要有保护，既要有社区又要有个体；

  4. 激励机制，以供应链业务为例，各方订单皆可进行融资，实现其利益诉求；

  5. 区块链是升级而非颠覆，是在原有系统上进行对接，让它在拓展能力等方面做得更好，而不是推翻原有的模式。正是应该通过金融科技的力量，把原有系统变得效率更高成本更低。

信任网络：通过信任来提高业务效率，降低拓展成本，增强监管能力尤其是增加事前监管，最终建立分布式商业基础。

答疑解惑

学员：Solidity算法比较新也不算很完善，用的人也不多，为什么会支持这个语言

李伟：这是第一个在以太坊上使用的语言，我们面对的很多客户都是之前在以太坊上做的原型或者智能合约。以太坊做互联网应用是没有问题的，但做金融应用不合适，所以我们要无缝接入以太坊上的智能合约。

学员：智能合约安全问题最近高发，代码比较复杂，你们是怎么做的？

李伟：只要是代码就会有BUG，要做到百分百无bug是不现实的，但可以降低概率。金融机构的智能合约，在发布审核上都十分严格，线下方面银行的风控合规流程都会走；线上方面，会有一些语法检查器或逻辑检查器，从技术上可以通过编程语言上的分析，检查基本的死循环和漏洞。部分中心化的设计也是出于这样的考虑，万一出问题可以有所补救。

学员：Solidity用起来有什么困难？

李伟：对JavaScript比较熟的话，Solidity难度不大，不过还是有一定的语法限制。

学员：技术方面趣链在做什么研究？

李伟：一是硬件加速，区块链和硬件的绑定会越来越紧密，已经初有成效，倍数明显；二是可信计算；三是公链POW、POS概率一致性存在问题，而联盟链主要是BFT算法，是有大量网络开销的，各有优缺点，是否存在一种算法同时资源消耗不高，又是确定的，又对网络规模是适用的？我们正研究新的一种联盟链和公链的共识算法，为了和未来物联网结合起来，微服务架构也会进一步细化。

学员：怎么看待玩客币这类产物？

李伟：这里要提到分布式CDN。现在大家看到共享单车这些其实都是租赁经济，而不是真正的共享经济。真正的共享是类似于Airbnb，还有流量共享。迅雷这就是把大家的闲置带宽聚集到一起，来提供网络加速服务，而其中上传下载所耗流量需要分布式线上结算：CDN服务的获得者付费，提供者收费，以玩客币结算，这就是激励机制，是个非常好的idea。但技术上来讲有困难的，需要有公链和联盟链结合解决，这就是区块链的魅力所在，其前程之广阔可涉及各行各业。
