银联支付页面报错1000005、1000098、100061、发卡行交易权限等的解释

  • 柚子社区
版权声明：本文为博主原创文章，转载请注明出处。 https://blog.csdn.net/u010882234/article/details/59483417

           银联支付，是国内比较成熟的一款支付产品。目前我们的项目，用的支付方式就是银联支付。由于我们用户量比较大，用户充值比较频繁，在银联支付的实际使用中，我们接收到用户投诉的各种各样的问题。如“交易失败，详情请咨询您的发卡行[1000098]”、“发卡行交易权限受限，详情请咨询您的发卡行[1000061]”、“订单金额超过该商户限额[5101405]”、“银联风险受限[2020023]”等，今天总结一下。

          1、银联风险受限[2020023]，发卡行交易权限受限，详情请咨询您的发卡行[1000061]，交易失败，详情请咨询您的发卡行[1000098]等

           一般都是交易金额受限。每个银行都有单日限额和单笔限额，银联也有单笔限额和单日限额，如果支付金额比较大，超过了单日或单笔限额，就会报类似的错误。这种问题，中国农业银行的卡出现的最频繁，银行农行单日和单笔限额都比较低，单笔2000，单日1万，超过就不能用银联支付了。

          而且，还有一种情况，当银行的交易量比较大时，达到他们系统的处理上限，从银联发往银行的支付申请也会超时失败。国内某些银行的系统，感觉需要升级优化下。我们曾接到大量投诉充值失败，时间大概有1小时左右。打电话给银联客服，银联回应这是来自发卡行的问题，支付申请没有按时应答，没办法处理，真的很坑。



          2、持卡人身份信息、手机号或CVN2输入不正确，验证失败[1000005]

           这种情况，大多是手机号的问题，银联的客户信息的手机号与发卡行预留手机号码不同，会导致这个报错。再说一下农行卡，也是投诉比较多的，农行借记卡有两个手机号系统，需要确定的是客户信息里预留的手机号，而非发短信提示的手机号。

           对于信用卡，CVN2有效期输入错误，也会报这个错。

           前段时间，我们的大量用户投诉，中国银行的卡，也报这个错误，最终原因让人无语。银联回复说，是由于银联系统与中国银行系统之间的通信问题，中国银行关闭与银联交互的老的接口。像这种接口的替换，难道不应该做到平滑升级，不影响线上用户吗？无语。


           3、银联官方查询地址

           这些出错，在网上都搜不到，很少有解释这些报错的。对于我们的系统而言，调用银联接口成功，跳转到银联页面，下面支付流程完全交给银联了。但是报错了，支付不了，用户可不管是谁的错，我们的客服没少因为这个向用户道歉解释。对于银联来说，也很委屈，好多结果都是发卡行返回的，对方不按时应答，也无可奈何。银联官网有关于应答码的解释：https://open.unionpay.com/ajweb/help/respCode/respCodeList。针对不同的报错，都可以去银联这里查询，虽然不一定能解决，但知道大概的问题方向，有个解释。其实当在银联支付失败时，最简单快捷的方法，就是用户主动打电话咨询发卡行，一般都能得到解答和解决。

补充：中国银联商户服务门户-帮助中心：https://open.unionpay.com/ajweb/help/query
