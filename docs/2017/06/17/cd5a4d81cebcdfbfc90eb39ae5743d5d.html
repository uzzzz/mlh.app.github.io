EMV规范（六）——处理限制

  • 柚子社区

处理限制的目的是检查终端应用和卡片应用的兼容性。结果记录在TVR中。

处理限制分为三部分：应用版本号检查，应用用途检查和应用有效期检查。

一.应用版本号检查

卡片返回的应用版本号的tag为“9F08”；

终端的应用版本号的tag为“9F09”。

若卡片未返回tag“9F08”，则终端应该假设卡片的应用版本号与终端的是兼容的，交易继续。若卡片的应用版本号和终端的应用版本号不一致，则应该置TVR的第2个字节的8位为1。

简单的说下，当the Application Usage Control 和 Issuer Country Code（tag"5F28"）都存在时，会去检查如下表的内容，当检查失败的时候，会将TVR的第2字节第5位置1。

三.应用有效期检查

有效期检查就是当前时间与卡片返回的应用有效期比较，看当前时间是否在有效期内。卡片返回的有效期有两个，有效期到期日期(Application Expiration Date的tag"5F24")和开始日期(Application Effective Date的tag"5F25")。若不是在此期间内，则TVR置相应的位为1。

 

总之，就是检查卡片返回的数据和应用是否匹配，若不匹配就设置TVR中的对应位。所以当交易的结果与预想不一致时，通常会看TVR来判断可能的哪些因素导致与预期结果不一致。
