2.区块链该看哪些会议论文

  • 柚子社区

发这期的主要目的是因为这次去武大开会，遇一武大研一学子，寒暄一番后，竟然发现他已然发了一篇A和一篇B的会议，其中一篇被收录mobicom，这是无线网络领域的顶会，一年才收30余篇，今年5月还投了CCS（信息安全四大顶会之一）。

纳尼？为何如此之强，我想和自己努力，还有找对方法有关，他说他只看2个顶会，看着看着idea也就有了。所以我们这期讲讲区块链该看什么会议。


信息安全四大会议：

S&P，全称：IEEE Symposium on Security and Privacy 每年11月截稿，第二年5月在美国开会,算是安全方向最好的会议。由于每年都是在美帝Oakland举办，会议也俗称Oakland. 这个会议接收的paper都是全面型的，好的idea 和 design，又有完整的implementation和好的效果，难度是四大中最难的。SP前两年的区块链文章还蛮多的，zcoin和zcash都是投于此，但是16年好像就中了一篇blockchain的文章。

CCS，全称：Computer and Communications Security 每年5月截稿，同年10月在美国开会。会议接收的paper偏理论和证明，不要求太强的实现，但是idea和design是要极好的。这个会议给了区块链单独的日程，16年有大概6篇的样子（包括智能合约）。大家可以多看看这个会。

USENIX，全称 Security Usenix Security Symposium 每年2月截稿，同年8月在美国开会。接收的paper偏系统类型的，漂亮的实现很受该会的喜欢。这个会收录区块链文章也比较少，16年也就一篇。

NDSS，全称：Network and Distributed System Security 每年8月截稿，第二年2月在美国开会，是四大当中相对最容易的，侧重新想法与实现。我找了找没找到任何区块链论文，因为人家是网络和分布式系统安全嘛。


区块链投的最多的会议：FC  Financial Cryptography and Data Security

FC 由国际金融密码协会 （IFCA）资助，主要接受与金融密码学相关的文章。虽然被CCF评为C类会议，但是他的质量并不差，他的出发点和区块链的正好一致，所以很多的文章投在这里。


 三大密码会（美密，欧密，亚密）：其中欧密这两年也接受了几篇blockchain相关论文。


总结：大家多关注S&P，CCS，FC会议中的论文。
