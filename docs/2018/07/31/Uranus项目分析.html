一、市场分析

Uranus，旨在成为计算机算力资源的uber和airbnb。

共享经济并非放之四海而皆准，滴滴历经生死大战而存活，共享充电宝死了又活，共享雨伞则彻底凉凉。为什么？

共享经济是有三种成功模式：

  1. 使用权经济，将物品的使用权暂时转移的一种经济模式，本质是使用而不占有，而物品的提供者可以是使用者，也可以是企业。
  2. 闲置经济，本质是为了物尽其用，由于互联网和物流的发达，人与人链接方便，无须见面就可以进行远距离交易。
  3. 慕课经济，利用了网络的便利性，突破传统教育的时间和空间的限制，一次录制N次学习。

共享经济的核心是最大化资源利用率以及可持续的盈利模式。

Uranus其实有些类似共享经济中的闲置经济，将全球闲置计算机算力资源集中起来，物尽其用，将其配置给有需要的人。闲置的计算机本身有电费成本需要覆盖，所以面临的问题就是需要足够多的用户需求，让算力资源有较高的使用频率。

Uranus利用区块链的撮合，将分散的社会资源，通过智能化匹配实现使用权分享，进而更高效地满足多样化需求。但是如何建立自己的壁垒呢？

如何引导用户加入泛在云？如何从传统公有云PK中获取自己的优势？共享算力服务本身是否比现有的技术方案更加稳定，落地后的运算性能能否达到日常业务的需求标准，这些都是需要时间解答的问题。

三、产品分析

泛在算力共享（Uranus，简称泛在）是算力资源里的Uber和Airbnb平台，为使用者提供高效率、低成本、去中心化的计算服务，是一个全球化的计算资源平台。

用区块链赋能社会冗余算力，以提供公有云（亚马逊、微软、阿里云）以外的泛在共享计算服务，从而重构IT算力市场格局。

用容器整合基于分布式计算的各种应用，以构建基于公链的生态圈。项目开发周期为一年。

针对目前计算资源的冗余情况，Uranus给出了如下解决方案：

针对各服务器资源冗余，市场协同和合作困难方面，利用区块链连接冗余算力端，从而进行高效、安全、可信的价值传递；

构建冗余算力端服务传递通道；
构建内容丰富的智能合约；
构建算力容器；
改造现有分布式云原生架构；
建立完善的应用商店；
自由矿池以承载价值；
创新分析：

Uranus链共识模块部分是在基于代议制权益证明机制DPOS和参考具有拜占庭容错的Tendermint算法之上，创新的提出DPOS+BFT混合共识算法。

“Tendermint 开源项目诞生于2014年，旨在解决比特币工作量证明共识（POW）算法中存在的速度、可扩展性和环境问题。通过使用和改进1988年在麻省理工学院开发的经过验证的BFT算法，Tendermint团队率先概念性地证明了某种权益证明（POS）加密货币的安全性，以解决第一代权益证明（POS）加密货币存在的「无利害关系」问题，如NXT和BitShares1.0”。

不过Tendermint采用的是担保交易，通过抵押债券保证金来作为安全抵押，而债券保证金会在一段时间里慢慢解锁，一般是2-3个月，这样做的目的是为了让轻节点用户有时间获得验证着名单的改变情况。

而EOS中不存在这样的经济惩罚，EOS 是委托人在失责的情况下将失去声誉来作为“抵押物”，破坏者几乎没有作恶的动机。

综上得出，基于 Tendermint 的 DPOS+PBFT 算法可能同 BM 提出的算法有不同之处，但 DPOS + PBFT 算法并非该项目的技术创新。

代币分析：

根据白皮书中的描述情况，首先 token 可以作为激励限制资源提供算力的矿工，奖励矿工提供算力服务。而使用算力的用户将通过 token 来支付其算力服务。

结合应用生态，用户量和交易量三者将成螺旋式上升，代币用途描述比较清晰。

综合情况来看，Uranus 的 token 结合了公链和应用链的特性。

升值逻辑：

Uranus 并没有提出燃耗机制和回购机制，作为公链在后期孵化应用生态的过程，很有可能作为私募来使用，该点为项目的代币升值逻辑。

青水评级总结：

算力共享的思路很正确，全球的电子设备闲置率惊人，造成了极大的资源浪费，如果这些资源能得到有效的利用，设备的拥有者，使用者都是受益方。

从整体上来看，项目整体落地仍然存在很多不确定性，路线图的规划仅仅一年时间，真实性有待考察。token 的设计较好的和项目整体的运行融合在一起，促进整个生态的发展。

代币的升值必定要伴随项目的落地和生态的繁荣，算力提供的稳定性和安全性是重点，理论上看对算力要求较高的行业和应用都可以考虑 Uranus 提供的算力服务。

青水评级总结：
总结一下，团队和顾问团队能力背景确实比较强大，曾任职大公司的不在少数，区块链经验也比较足，开发人员多，后期一旦发力，势不可挡。

青水评级综合评价：

1、Uranus通过引入区块链的方式，充分利用了目前市场上的闲置资源，并且通过去中心化的方式，解决了现有公有云付费方式不灵活、资源利用不充分、各项服务不稳定等问题。

算力共享思路很正确，愿望是美好的，愿景是远大的。市场也给与了一定的回应。项目开发难度大，落地仍然存在很大的不确定性。

2、Uranus的通证由算力共享量和数字函数产生，自有矿池中存储一部分通证供买家购买，买家获得通证后可交换相当量的算力。作为激励服务提供者拿出更多更好的算力容器加入网络，Uranus增加了在线时长证明机制和算力证明机制，以完备 Uranus 的激励机制。

通证这块在白皮书和官网中没有披露更多的信息。从白皮书整体来看，Uranus的通证更多是在生态系统的所有参与者能够共享或购买计算能力并提高交付价值。通过PoC（贡献证明）机制，促使服务提供者提供更多计算能力的机器节点来获得更多通证的激励。

3、项目融合了公有云技术和区块链技术，具有很强的技术挑战性。官方和另外一个项目oVirt有着千丝万缕的联系，但是官方没有披露这一细节，诚信有待商榷。不过顾问团队确实强大，相关区块链经验很丰富，或许有一战之力。

目前，Uranus还处于预热阶段，其他情况有待官方进一步公布。
