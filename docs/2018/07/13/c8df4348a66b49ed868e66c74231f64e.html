ETH? EOS? 你怎么选？

 

之前我们讨论过，此次以太坊“燃料危机”的成因及其带来的影响，（没看过？送你一道任意门《以太坊空前“Gas危机”大爆发，而真正遭殃的是这些Dapps》），除了再一次爆出以太坊网络本身存在的问题，同时也牵连了一批Dapp及其开发团队：推迟上线，挽回用户，他们甚至已经开始布局，将Dapp向其他公链迁移。

 

01

迁移到EOS上的UNICO

将在加密收藏领域创造价值

https://www.unico.global/

 

月初，UNICO官方宣布称UNICO已迁移到EOS上建立Dapp。UNICO的功能是创作者借助它可将高质量的数字作品如照片、视频、3D模型、VR场景等等，转化为限量版的加密收藏品，并出售。粉丝和收藏家可以在高度安全的EOS生态系统中购买和交易有价值的收藏品。

 

UNICO团队在Medium上发表相关文章，公布UNICO已正式发布，同时也提及迁移到EOS是“经过多次的讨论和对比后，得出EOS是UNICO的最佳选择。”其CTO Ivan提到“EOS无手续费、可拓展性和强大的安全性能等特性是吸引很多开发者的主要原因，早在两个月前，我们就已经着手迁移了。同时UNICO项目也受到了EOS生态社区成员的关注和支持，我们很高兴并在不断努力完善UNICO的更多功能和扩展其应用价值。”

 

 

02

ProChain抢占先机，

打造EOS生态上首个广告服务

https://www.prochain.com/

 

此外，区块链广告行业的引领者ProChain也计划将广告业务从以太坊横向拓展至EOS。ProChain(PRA)作为发力区块链广告行业的知名项目，早已凭借安卓和iOS版本的区块链应用Prabox在愈发激烈的市场竞争中占得先机。同时创始团队密切关注着EOS超级节点竞选及EOS主网上线的市场热度，预感到EOS公链上会诞生数量可观的Dapps，并将目光瞄准了EOS。

 

ProChain其创始人老白在接受采访时说道：“以太坊存在的拥堵问题人尽皆知，我们希望通过公链拓展的方式为用户们带来更好的产品体验，同时为更多EOS初创项目提供区块链广告服务。”

 

高交易速度及免费交易吸引ProChain生态向EOS迁移，且EOS支持高并发交易，这已经是对现有大多数公链的超越。此外，EOS有30多万人的社群、有数百个优秀开发团队支持，这些优势就足以吸引ProChain以EOS为基础创建自己的生态，并为该生态服务。

 

 

03

搭上EOS快车的Everipedia

会是下一个维基百科挑战者吗

http://everipedia.org/

 

 

2017年末，Everipedia团队的一则公告让这个名字重新走到区块链世界的聚光灯下。维基百科前联合创始人Larry Sanger的加盟，同时Everipedia将把审核文章，编辑和存储信息的整个流程迁移到EOS区块链上并发行名为IQ的代币，向着超越维基百科，这个世界第五大流量网站的目标全力进军。

 

Everipedia团队认为，使用智能合约来保存编辑工作流程和文章，同时借助一套真正的审核系统，将Everipedia打造成为一个信息更准确的百科平台。通过发行代币来激励内容贡献者，让原创内容得到保护，同时能吸引到更多的内容创作者加入进来，形成一个良性的生态圈。

 

目前，Everipedia有600万页面，230万独立用户，但相较于维基百科的4300万页面，1亿3千6百万活跃用户还远远不够。虽然还有很长的路要走，但Larry Sanger表示“对于百科全书的一次新的探索和进化，正是我们团队在做的事。当信息真正实现去中心化，并由运行在区块链上的智能合约提供时，它能为人们开启新世界的大门。”



Dapp迁移到EOS，

远比你想的要简单

无论是在TPS处理速度上、可拓展性、手续费用及安全性能，EOS都更受开发者青睐，那么已经在以太坊上运行的Dapp，如何迁移到EOS上呢？其实不难，EOS脚本和虚拟机了解一下。

 

- EOS支持多种虚拟机-

1.EOS平台

在白皮书中，EOS首先给自己的定位是一个“平台”，用于协调向账户传送已认证的消息。而具体的脚本语言和虚拟机的细节都是特定于实现细节的，这些细节大多数独立于EOS的技术设计。因此EOS API 可以集成任何具有足够性能的确定性和正确的沙箱化的语言或虚拟机。

大白话：大的技术架构交给EOS，那些具体实现的细节，语言、虚拟机自己弄，完了跟EOS对接就行。

 

2. 认识EOS架构

架构定义的消息和数据库：

消息：在账户之间发送的所有操作都是通过属于区块链共识状态的架构来定义的。该架构允许在二进制和JSON表示形式之间进行无缝转换。

 

数据库：数据库状态也使用类似的架构进行定义。

即账户间消息的发送是二进制，数据库的数据存储同样是二进制。因为计算机中的数据都是0和1的二进制指令，直接执行二进制指令，减少计算多余的性能损耗。而EOS也能将二进制的数据转换成人类可读的JSON格式。

 

多索引数据库API（白皮书2.0新增）

开发智能合约需要定义的数据库架构来跟踪、存储和查找数据。开发者通常需要对多个字段进行排序或索引相同数据，并保持所有索引之间的一致性。也就是开发人员在查询庞大的数据库数据时，EOS给他们提供查询的入口，并支持多个字段查询。

 

分离消息认证与应用

EOS通过并行执行机制来实现高性能，为了高性能，EOS无处不在考虑怎样进行并行和串行的区分。

 

EOS为了减少从程序日志中重新生成应用程序状态的计算任务，来最大限度的实现并行计算，EOS将验证逻辑分成三部分：验证消息是否内部一致；验证所有先决条件都是有效的；修改应用程序状态

 

分离：消息认证是一个验证消息可被使用的只读过程，而修改应用程序状态是写的操作，EOS将它俩分开。 而且一旦消息被包含进区块就不再需要进行消息验证的操作了。

 

3. 虚拟机独立架构

EOS的架构是独立于虚拟机的，理论上只要符合性能、确定性、正确性和沙箱化这几个条件的任何虚拟机都可以跟EOS进行对接。

目前公布支持的有三个虚拟机：

Wren、Web Assembly【web 组件】(WASM)和以太虚拟机（EVM）

 

在这里重点讲一下EVM。EVM是以太坊智能合约的运行环境，以太坊的核心组件。在EOS区块链系统上，EVM合约能在内部沙箱中运行，只需要少量适配就可以与其他EOS应用程序交互。这样以太坊上的智能合约就可以运行在EOS上，迁移就很容易实现了。

 

随着区块链生态的日渐完善，Dapp的迁移和流动未来将成为一股潮流，最终经过无数人的选择，推动真正有价值的Dapp融入到大众并触达更广阔的应用场景。其实，真正成功的区块链分布式应用，不是让人记住它基于EOS或是ETH，而是Dapp本身。
