总结 第一次pat甲级考试[18 冬季]

  • 柚子社区

考前准备

考试两个月之前间断刷过15道题, 每道题平均都会花一天的时间才能ac掉, 当时就想的为pat准备, 但是计划赶不上变化, 因为其他的原因放下了pat甲级题的跟进, 不过就那些时间也收获颇多.

题目

这次的题目相对于甲级题库的题来说还是相对简单的, 每道题大概得花了3分钟看题, 半分钟出现大概思路, 10分钟之内会出现具体的数据结构与算法, 其他时间都用来实现和改bug了.
大概说下记忆中的题目吧, 题目为我理解的大意:

1. 求素数(20分)

给出了一串数字, 长度为L, 给出一个你需要求的素数长度K, 要你在L中找到长为K 的素数.
要注意的是, 类似于 00002 这样的数字, 打印时要同时给 0 也打印出来(我似乎因为这里没注意到扣了2分)

2. pat考试信息查询(25分)

每个人的信息由两部分构成 (信息 分数),
其中信息由四部分构成:

  1. 第0个字符, TYPE, 此次考试的类型
  2. 第[1,4)个字符, site, 101-999
  3. 第[4, 10) 个字符: date, 参加考试的日期 yymmdd
  4. 第[10, 12) 个字符: number, 考生的number, 000-999

查询有三种可能:

  1. 查 type, 输出该type的所有信息, 按照分数排序, 分数相等, 按照学号排序
  2. 查site, 总计该 site 由多少个人, 以及所有人的总分
  3. 查date, 总计该日期的每一个 site 有多少人, 按照人数排序, 人数相等, 按照site编号排序

3. 判断该种图着色是否可行(25)

给出一个图, 并给每个节点着色, 判断这种着色方案是否可行(相连的节点颜色不能相同)

4. 似乎跟一个堆有关, 没时间看了(30)

总结

失败的原因在于:
时间问题, 因为编译器的一个函数小问题, 第一道题花了1个小时, 用c跟java都写了一遍…第二题花了1个半小时左右, 先用c写了一半发现这个编译器问题后没法调试换成java. 这样就只剩下30分钟来做后面两个题了…导致第三题数据结构设计过于简单, 超时了…
对语言还不熟悉: 导致出现了一些意想不到的bug
出现很多超时的案例…下次还是学着用c++吧…
归咎原因, 还是刷题不够…
