EOS环境搭建 + 自带合约介绍 + HelloWorld合约（1）

 

一、准备条件

①Vmware WorkstationPro

②Ubuntu16.04（创建虚拟机的时候，内存不够8G的话，4G也可以）

 

二、坑逼之路

1.Alt+Ctrl+t 可以打开终端。

2.在终端中下载git。输入git回车，他会提醒你如何下载git。

3.git下载好后，记得git init，创建本地仓库。

4.下载源码:git clone https://github.com/EOSIO/eos --recursive

5.此时你已经下载好了源码，并且将其放在了eos文件夹下,所以 cd eos

6.如果虚拟机内存小于8G，别着急启动编译，先去修改脚本绕过这个限制。

方法：（1）eos文件夹-->scripts文件夹-->打开eosio_build_ubuntu.sh文件

        （2）找到如下图位置，并将exit 1前面加上#号，保存。代表注注销掉这句话。

7.进入eosio_build.sh中的CMAKE_BUILD_TYPE=Release改成CMAKE_BUILD_TYPE=Debug

8.返回到刚才的终端中。输入命令 ./eosio_build.sh

这里开始，即将巨坑无比！此命令会编译所下载的源码，并且下载许多支持EOS的资源库。其中会出现问题主要为：下载 Boost1.67 或者 MongoDB 或者 MongoDB Driver的时候出现不下载的情况。凡是遇到这种问题，我是Ctrl+C推出后，再次./eosio_build.sh，甚至多次，就解决了。如果想自己下载这些资源库，请核对好版本，版本差一点都不行。

9.编译时间较长，结束后会出现 EOSIO几个大字儿。然后请安装，命令 sudo make install

10.cd eos/build/programs/nodeos 跳转到nodeos下,运行下面的代码来启动节点

11.输入以后就开始出块了

12.如果在启动节点的时候的有错，则是可能EOS版本的问题，链的配置文件多少会有些出入，那就更换或者添加一些东西在配置文件中即可。cd home/你的用户名/.local/share/eosio/nodeos/config

vim config.ini。填一下这些内容，或者你自己去EOS的github上找官方提供的也行。
