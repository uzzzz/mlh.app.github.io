17年碰到的几种离职场景

时间还是蛮快的，17年在上半场的忙碌和下半场的悠闲中度过了。
这一年的收获主要是在管理层面上的进步，从最初的赶鸭子上架到现在的游刃有余也算是收获吧

过程中有人离开有新人加入，作为管理者/合作伙伴/朋友得出的情况有以下几种

  • 钱不到位走了
  • 工作太繁忙/太松弛走了
  • 发展方向不适合走了

以上三种离开理由其实除了第一条我觉得都是可以谈的

劳逸结合确实是需要的，阶段性的繁忙是很正常的，加班对于这个行业来说已经是正常事情了（当然从入行之初到现在我都认为默认加班这个思维是很不合理的，但是实际情况有时候不得不强奸我们一回）

但是长期处于高强度，极度紧凑的发版周期对工作者的身心健康其实都有影响。所以阶段性的繁忙和高强度我觉得还是能接受的，如果张嘴“我就不加班！“，当然这样的企业也有，但是大部分互联网企业估计是呆不久了。

刚才也提到阶段性的繁忙，那就有阶段性的“清闲“。但其实清闲不清闲取决于自己给自己的压力与要求，见过1年学3门语言入门，1门深入的。见过闲3个月封装一套组件都没进展的，问其回答却是“我不知道怎么做！“

看穿不说穿，很多时候也能感觉到有些小伙伴在闲暇时愿意和其他人一起做一些积累，也有空下来看“大军是司马懿“的，路都是自己的，只是都是自己的，时间也是自己的。懒惰和勤劳在两年就会有天壤之别，所以这一类离职理由我都是放行，祝君一轮顺风那种！

除非你身处独角兽的高位，不然一定不要放弃持续学习！

我手里的小伙伴都是前端+app开发的，因为我个人对于技术的理解属于“技能只是工具，不要因为工具而限制了产品的更好发展“。所以我希望小伙伴们熟练掌握“前端“相关的所有基本技能，以便于任务来到时不会那么不知所措。

但是也遇到过两种：

  1. “我是面试RN的，做移动APP，Web的React我不想做“
  2. “好的，这部分我去看下，尽快学会掌握“

也是这两类人，不同的选择，会给管理者完全不同的印象。
所以给机会学习，而拒绝的一部分小伙伴，走的话也是放行，祝君一轮顺风

2018的一些小目标

区块链技术

这两年区块链大热，但是大多数人只是在一虚拟货币，却没有关注技术本身的价值。是否这些理念应用会实现落并大放异彩，我会持续关注

微信小程序
在刚推出之初饱受诟病，直至在17年3 4季度大放异彩，没有分第一杯羹是自己的失策，所以要奋起直追，进一步深化相关知识，相当看好这个市场。

Python
虽然公司的主体实现还是Java的，但是语言本身的缺陷也很明显，在后一代的高级语言里还是喜欢Python，简单好用虽然之前就会，但是一直没有深入的试探，今年希望有所收获。

好用的CMS系统
几乎所有运维场景都需要这么个东西，一直没有很满意的实现结果，虽然手里有但是总没有达到心里那种随心所欲的快感，也希望今年实现一个能然自己满意的CMS系统吧。
