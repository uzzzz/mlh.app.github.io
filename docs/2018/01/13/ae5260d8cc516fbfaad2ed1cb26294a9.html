如何编写智能合约之三：使用OpenZepplin函数库开发智能合约

  • 柚子社区

前面有两节笔记已经说过truffle的相关情况，这里就不再赘述。\
本笔记是在truffle的基础上，使用OpenZeppelin函数库来创建一个简易发币合约。OpenZeppelin可以简化我们的开发。因为它内置了很多东西，包括安全等方面。

PS： 我在安装的时候，遇到了一个问题。就是上面的步骤执行完成后，package.json文件并没有增加dependencies选项。本来以为没有什么问题，手动增加上去就可以了。但是在编译合约的时候，就出现问题了。显示我引入zeppelin-solidity包中的文件不存在。各种找问题，但是我尝试了网络上的所有关于引入包文件不存在的问题的解决方案，均不能解决。尝试在引入文件的时候，使用相对路径，但是并不能解决。就去看以太坊官方文档中关于包文件引入的方法，对比了自己代码，并没有问题。各种方法尝试后，均没能解决。就尝试了更新npm。更新完成后在重新初始化开发环境，确实可以了。

  1. 编写合约BLCoin.sol,合约内容：
pragma solidity ^0.4.18 ;
import "zeppelin-solidity/contracts/token/StandardToken.sol";//这句代码在atom中还是会报无法找到文件的错

contract BLCoin is StandardToken{
  string public name = "BLCoin";//代币名称
  string public symbol = "BLC";//代币简写名
  uint8 public decimals = 4;//代币的最大单位和最小单位之间进制是10的4次方
  uint256 public INITIAL_SUPPLY = 10000;//代币发行总量

  //初始化
  function BLCoin() public {
    totalSupply = INITIAL_SUPPLY;
    balances[msg.sender] = INITIAL_SUPPLY;
  }
}
