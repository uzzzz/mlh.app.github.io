如何让你的硬盘躺着也能赚取空间分享与PoS“双重奖励”？

  • 柚子社区

硬盘也能挖矿赚取奖励？

Genaro Sharer 回答你，是的！

硬盘挖矿就是通过将自己的硬盘空间分享出来而获得奖励。

Genaro Sharer 虽然在严格意义上讲不能称之为挖矿，但随着“硬盘挖矿”等新概念的出现，为了表述方便，我们将分享资源（包括算力、存储、带宽等计算机资源）获得奖励的行为统称为“挖矿”。 

Genaro Sharer 的挖矿不同于比特币或以太坊的挖矿，这类挖矿对计算性能要求极高，还要不停地计算哈希，普通用户几乎不可能挖到。相比之下，Genaro Sharer 的分享门槛低，更加亲民, 还能获得“双重奖励”！让有条件的人都可以近距离体验区块链技术带来的魅力。

总的来说，过程可以分为两部分：

- 第一步：进行 Stake（权益放置）

- 第二步：分享硬盘空间以赢得奖励

 

▼

详细教程如下

对于挖矿有两种方式，首先需要向你的网络运营服务商得到一个公网IP。

 

01

Genaro Sharer 客户端（Mac/Windows）

下载 Genaro Sharer 客户端, 并确保钱包内已有足够调用智能合约的 ETH 和足够 Stake的 GNX 数量。

1. 分享硬盘空间开始获取奖励

进入“我的硬盘”界面，点击右上角“分享硬盘”。

选择需要分享的硬盘位置与容量。设置完成后，点击“确定”。

分享成功后可以在“我的硬盘”管理列表中查看。若硬盘分享成功则显示绿色状态，失败显示红色状态。

2. 设置 Stake 钱包

点击“更多”按钮，选择其中的“Stake”可设置 stake 钱包。每个硬盘都可自定义选择已在Genaro 钱包管理中添加的钱包为此分享硬盘的 stake 钱包。

您可以在设置Stake钱包输入框中，填入 GNX Stake（权益放置）量（最低限制为：5000 GNX）与其锁定时间，输入钱包密码后确定设置。硬盘分享奖励每24小时更新一次，您可通过第三方钱包管理平台查看每日收获记录。

3. 查看分享日志

点击“Show Log”以后会打开日志目录，可以通过看日志的方式来判断分析分享状态。

4. 查看Stake钱包与GNX 奖励信息

点击“Show Reward“后可以显示 stake 钱包，以及目前分享收获的 GNX。

5. 提取收获的 GNX

在上个步骤中可以看到自己的奖励，点击提取，将奖励转账到 stake 钱包。如图，转账时会得到一个转账的 hash ，通过 etherscan.io 查看转账进度。可能耗时比较长，取决于当前网络拥堵状态。

以下为转账成功的状态。

 

02

Linux版本

1. 进入命令行

Window - 开始菜单 - 输入cmd;Mac - 找到实用工具 - 终端

2. 安装genaroshare-daemon

安装说明可查看： 

https://www.npmjs.com/package/genaroshare-daemon

3. 新建配置

在执行这个命令后，会生成或直接打开生成的 json 文件。懂配置的朋友可以更改其中的配置，这些配置都是高级设置，请在改之前确认自己明确了解自己改的是什么。

对高级设置不熟的话可以选择直接输入：

genaroshare-create - - name <你需要添加的账号（钱包）名字> - - noedit

这样会直接显示如下图：

这个步骤会生成一个新的钱包，请牢记您的助记词、 name 、密码以及私钥（private key），后续取 GNX 奖励时需要。这个钱包专门用于挖矿收益，以及 stake，建议一台机器只建一个配置文件。

下一步以这个生成的钱包进行 stake。上图所示的json文件，里面可以查找到具体的设置，对应的 d8aa0ace80a84776d7082f91d67e2a6bcf1a3a0f 是对应生成的存储网络ID。

* 助记词/私钥的使用：可以导入到其他钱包软件中，进行管理（转入转出等）

 

4. 修改默认配置

上一步骤会生成配置文件，这个文件中设定了分享的各项属性。有几项需要自定义修改：

- rpcAddress 后面的值从 127.0.0.1 改为机器的公网 IP

- doNotTraverseNat 的值改为 true (如果有公网 ip 的话)

- storagePath 的值改为分享的路径

- storageAllocation 的值改为需要分享空间大小，默认20G

 

5. 用户需要保证钱包里的 GNX 数量以执行后续操作

钱包地址请看生成的 json 文件，里面的 Payment Address 的值。

您需要保证这个钱包里面有足够的 GNX (大于 5000) 以及 ETH（0.0069395 ETH (20Gwei) - 0.01734875 (50Gwei)）

确保这个里面有 GNX 和  0.0069395 ETH (20Gwei) - 0.01734875 (50Gwei) 用来做燃料，因为 GNX 现在通过 ETH 上的 simulator 来进行矿的调度。现有版本使用 Ethereum上的 emulator 所以 block 的燃料现在由 eth 提供，在测试链和主链都上线的版本会以全 GNX 的方式实施出来。

 

6. 进行 Stake

只需要输入命令：

genaroshare stake --nodeID 070932e1a01a4820836e4d32754821088ff8352d --quantity 5000 --option 0

其中 nodeID 是之前对应生成的存储网络 ID，在这个里面是070932e1a01a4820836e4d32754821088ff8352d，quantity 是要分享 GNX 的数量，具体细节可以参考挖矿细则，option 是时间，其中0 代表30天，2 代表60 天，3 代表90 天，… 12 代表360天。

 

7. 开始分享

之后可以使用原有的连续命令 ：

1. genaroshare-daemon

2. genaroshare-start --config /Users/weilongwu/.config/genaroshare/configs/<指定的nodeid>.Json

来开启genaro node 以及 使用 genaroshare status 命令来查看状态，具体如图：

需要running，delta，port，和bridges的对应的值都为绿色才算连接成功。

常见问题

Q1：Genaro Sharer 双重奖励具体包括什么，如何分配？

Genaro Sharer 双重奖励分为存储奖励与 PoS 奖励，具体为：PoS 奖励为节点做 PoS，将 GNX 进行 Stake 所得奖励。主链上线前，Genaro Sharer 使用以太坊智能合约进行奖励分配，每年大约2%，以月为单位进行 GNX Stake，所获 GNX 线性释放，按天结算，奖励 GNX 来源于系统在线即有奖励。主链上线后，Stake 节点的奖励主要来源于系统奖励与出块奖励（transaction fee）， 具体细则等Genaro Network Testnet发布时再进行公布。

存储奖励目前为贡献空间所得奖励，计算方式：1 GB每年0.1 GNX，1 TB每年102.4 GNX，1 PB每年104857.6 GNX，具体奖励取决于实际用户的使用量。例如分享1 PB，但实际用户一年使用平均值为1 TB，则奖励为102.4 GNX。之后，Genaro Sharer考虑在存储奖励部分增加带宽分享奖励，暂定方案为：1 GB/0.2 GNX，1 TB/204.8 GNX，1 PB/209715.2 GNX，具体奖励取决于实际用户的下载量，目前的版本尚不支持带宽奖励，需要等到之后的版本。需要注意的是，节点必须先进行PoS Stake，才可以分享存储空间，也即无法单独只获得存储奖励。

 

Q2：Genaro 从中收取费用吗？

Genaro 从中不收取任何费用，Genaro Eden 与 Genaro Sharer 是一个完全的共享经济社区，用户使用存储空间、带宽、消耗 gas 全部分配给相应的分享者节点。

 

Q3：如果没有数据分配到我的存储节点，我还是否能获得奖励？

主链上线之前，您依然可以获得 PoS 奖励，任何人做 PoS 都可以获得2%奖励，主链上线之后，需要看节点是否被选进委员会等情况来判定。但如果所分享的存储空间没有被用户使用，则无法获得存储奖励，不过系统会进行在线即有奖励，根据节点的发展情况进行分配，也有获得这部分奖励的可能。

 

Q4：PoS Stake 需要放置多久，可否可撤回？

节点可以选择 Stake 1-12个月。主链上线之前，一旦 Stake，无法撤回，需要等线性释放完毕，主链上线之后，可以撤回，但会有相应的惩罚机制。主链预计2018年Q4上线，如果节点的 Stake 时间晚于主链上线，Genaro Network 映射时会为节点进行特殊映射，进行相等时间 Stake 的准确结算，从而保证通证的流动性与公平性。

 

Q5：<GNX allocation statement> 中在线即有奖励的细则是什么？

GNX 中总量的5%，即3375万 GNX 作为在线即有奖励，在未来的5年内分配给 GNX 生态中的分享者，包括补贴使用者免费空间与带宽的使用以及分享者 PoS 奖励，发完为止。

 

Q6:  挖矿对带宽或者硬盘有没有要求？

现在主要是考虑到网络的稳定性，因此现阶段对于贡献空间的最低要求是4TB, 对应 PoS 所需 GNX 为5000个。对带宽没有硬性要求，但是带宽越高，越容易成为分享节点，并且带宽要视硬盘容量而定，建议带宽至少为10m，端口是4000开始，可以映射。

随着网络节点越来越多，我们会推出 Mining Pool（矿工池），不断降低挖矿门槛，使更多的用户参与进来。

 

Q7：如何成为稳定的节点？对持仓有没有什么要求？

保持在线即为稳定节点，持仓量越大对应可分享空间越大。

 

Q8：如果停止 Genaro Sharer 存储分享（人工暂停或者停电），是否会受到惩罚？

主链上线前： 无论是否进行分享，或将存储暂停/停止，都可以获得 PoS 奖励，Stake的 GNX 也同样安全。但是无法获得对应的存储奖励。

主链上线后：系统会对节点的表现进行分析与判断（包括主链上线前的表现），如果节点下线时间过长或响应较慢，会被视为恶意节点剔除，会对 Stake 进行一定的惩罚，将剩余部分自动归还。相反，如果节点表现优良，则会被选进委员会，并进一步可能被选作出块节点，获得更多的 PoS 奖励。细则需要等待 Genaro Network Testnet 上线后发布。

 

Q9：后期硬盘挖矿奖励和 PoS 奖励会分离吗？

目前的方案（无论主链上线还是现在基于以太坊的版本）是必须进行 PoS Stake 才能进行存储分享。主链之后会对奖励分离的解决方案进行探讨。

 

Q10：现在是否支持存放其他的 ERC-20 Token。转入其他的 ERC-20 Token 怎么办？怎么取回来？

Genaro Eden 与 Genaro Sharer 的钱包只能存放 GNX 与 ETH ，如果不慎将其他 Erc-20 token 转入，需要将钱包导出来，然后放进现在主流的 ETH 钱包( MEW, Parity )里即可。

 

Q11:  对于已经参加 PoS 试验计划的 GNX 还能进行 Stake 吗？

非常遗憾，不能。如果仍在在锁定期的 GNX 将不能进行新一轮 PoS 来获取奖励。不过，参加 PoS 试验计划锁定达8个月的用户所获系统的实际奖励高于目前挖矿的奖励，也是感谢从始至终的长期支持者。

 

Q12：团队能否推荐好的节点配置？Nas 做节点效率高还是电脑磁盘阵列效率高？

现在主流电脑都可以，因为 Genaro 不是一个很重 GPU 或者内存的挖矿计算方式。NAS 会效率高一些，主要要看个人带宽的大小了。
