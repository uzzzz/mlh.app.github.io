命令行下编译Qt程序

我用的是 windows 系统 + MinGw5.3.0 32 bit

  • 首先我们先新建一个文件夹，用记事本（自行选择）写一段代码 嘿嘿(记事本没有自动提示和补全)(●ˇ∀ˇ●)
    main.cpp
#include<QApplication>
#include<QWidget>
int main(int argc,char ** argv)
{
    QApplication app(argc,argv);
    QWidget* pWidget = new QWidget();
    pWidget->show();
    return app.exec();

}

文件夹下只有这一个 mian.cpp 文件

用到的工具 Qt 5.9.2 for Desktop (MinGW 5.3.0 32 bit)

  • 进入到相应的 文件夹下 输入qmake -project 会发现文件夹下生成了平台无关的项目文件一个XXX.pro文件(前面和文件夹名字一样)

  • 下一步输入 qmake testCompileQt.pro 你要输入对应的pro文件
  • 会生成一些 makefile 文件 和 debug 和 release 文件夹（因为手写Makefile是比较困难并且容易出错的，尤其是需要给不同的平台和编译器组合写几个Makefile。使用qmake，开发者创建一个简单的“项目”文件并且运行qmake生成适当的Makefile。qmake会注意所有的编译器和平台的依赖性，可以把开发者解放出来只关心他们的代码）

在windows上，如果使用的是开源版的Qt，用mingw32-make.exe；如果是商业版的Qt，用nmake

  • 所以我输入mingw32-make，会 报错 不要着急

因为我们用到了QWidget 所以要在pro文件里加入 QT += widgets 然后继续重复上一步

这次可执行文件生成了 并且成功执行
