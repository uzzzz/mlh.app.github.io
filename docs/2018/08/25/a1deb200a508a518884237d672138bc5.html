个人网站支付收款怎么解决【经验分享】

  • 柚子社区

微信、支付宝的收款业务，相信大家都很熟悉，找到相关页面自己的二维码就会显示，付款人扫一下就可实现收款，简单方便。然而我们这里所说的“个人收款”，不是普通的扫码服务，而是支持回调，例如在线商城支付后，商城可以知道支付状态并且自动修改订单的状态为“已支付”。这种支付的形式，目前无论是微信、支付宝还是银联，都是不对个人开放的，必须有企业资质才能申请，也就是我们平时付款时扫的“商家二维码”。这对想要实现支付功能，又没有企业资质，无法申请到支付宝微信等第三方支付接口的网站站长来说，是个不小的麻烦。

 

当然，这也不是完全解决不了的，都说“劳动人民是智慧的”，如今就有这么一群“码农”，为打破个人收款瓶颈，利用个人支付宝、微信自己实现支付功能的思路，只需一部旧手机，就可享受免签约、免费率、免提现的收款码。他们就是E交易定制式接口技术团队，深耕支付领域，为所有无法直接申请第三方支付接口的站长带来福音。

 

这个方案的基本思路是非常简单的，跟大家常用的爬虫爬取网页账单数据类似，我们这里用的是手机APP。相对来说，截取手机APP的推送消息更为简单，不需要应对微信支付宝的各种反爬措施，不过也有缺点，就是能够获取到的信息较少，没有诸如流水号、付款人之类的信息，只能看到付款金额,不过E交易的APP却能绑定订单号!完整实现订单流程并支持了高并发

 

E交易定制式接口支付操作非常简单——

 

1）创建一个订单，将二维码展示给用户，注意这里的二维码不需要用户事先上传更不用什么1.1,1.2,0.9式支付，是绑定跟踪订单号而非价格的，是APP帮您生成的二维码（这个很厉害！比目前市场上的都好用）

 

2）用户支付后，商家手机APP上会收到支付宝的支付宝/微信/QQ推送

 

3）安卓手机APP截取支付宝/微信/QQ的付款推送，然后将付款信息发送给服务器

 

4）服务器根据付款订单号，确定到底是哪一笔订单后将该订单标记为“已付款”，然后根据需要进行回调通知之类的操作。

 

E交易定制式接口支付不需要企业资质，可根据日账多少选择开户模式，一次性买断无需任何手续费，只需提供个人收款码，便可轻松享受商家收款体验，99.9%超高成功率，免人工操作，无任何行业限制。唯一需要注意的一点就是，需要有一部手机一直运行着，且要求网络条件良好，否则可能会造成支付数据丢失。

 

当然他们的支付的接口文档那些也很完善：类似的我们请求下面参数就能返回想要的二维码或者支付链接

					    
		{ 
			"app_id": "pCAUhH8SlFfWMy", 

			"order_amount": "1",

			"charset":"GBK",

			"order_no":"20171501147646468"

			"notify": "http://www.example.com/notify.php", 

			"type": "wx", 

			"mobile": "1", 

			"sign": "c81bbbb4810806a34f8e02246fd81341",

			"subject": "测试商品", 

			"time_expire": "1m" 
		}					      	        
					  

 

结果返回为

字段 必选 字段类型 说明
app_id true string(35) E交易平台支付系统分配给商户应用的唯一身份标识
state true string(10) 订单支付结果[默认 SUCCESS]
order_no true string(11) 支付订单号为 8-20 位 数字、字母或组合
subject false string(10) 支付订单商品名称
charset false string(10) 请求使用的编码格式[默认 UTF-8]
pay_url true string(10) 支付订单对应二维码链接或H5支付跳转链接[支付二维码链接需要用户自身转换为二维码]
 

 

当然你要用E交易订单式接口也可以，订单式的这里不做赘述，不需要挂机的也是秒到账的，但是灰黑色行业不能使用定制式支付接口
