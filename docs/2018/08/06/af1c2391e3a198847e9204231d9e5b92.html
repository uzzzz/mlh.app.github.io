自治，甲骨文继续领先的开始？

  • 柚子社区

亚马逊计划在2020年初完全弃用Oracle数据库！

正是在这个老孙朋友圈刷屏的新闻爆出的前提下，一年一度的甲骨文数据库云大会如期在北京召开！

这已经是甲骨文连续五届在中国举办类似会议。

很显然，面对亚马逊的“挑衅”，甲骨文早有准备。

“AWS一直想摆脱我们，说了几年了，可一直没有成功。他们最早的计费、财务分析系统和数据仓库、数据分析系统，其实一直在使用我们的产品。从2006年开始，亚马逊开始做云，于是变成了甲骨文的竞争对手之一。2009-2010年，他们就开始了从公司层面要去除甲骨文的数据库产品。然而这么多年了，到今天他们还是没有真正做到，也蛮尴尬的。祝他们好运，早日实现梦想。”甲骨文公司数据库技术执行副总裁 Andrew Mendelsohn在媒体沟通会上强硬的回应了这个问题。

甲骨文公司高级副总裁及中国区董事总经理李翰璋说，“去IOE已经说了很多年了。每年媒体都会问我看法，我的看法是他们的做法可以理解，有他们的道理，我们无法阻止。我们能做的事情就是，坚持用深入基因的创新来引领整个潮流的改变。甲骨文在数据库有40多年的经验，另外每年投入在研发当中有60多亿美元，其他还有额外的并购等等，这些都拉开了我们与竞争对手在数据科技方面的领先距离。”

自治——甲骨文持续领先的开始

正如李翰璋所说，甲骨文在数据科技方面的确一直领先于竞争对手，Andrew Mendelsohn如此解释甲骨文之所以能够持续领先的原因：“甲骨文在数据库领域的领先优势源于Oracle数据库与时俱进、持续创新的高度活力。”

而这种与时俱进、持续创新从1998年Larry Ellison创建Oracle数据库开始就一直延续至今，“Oracle自治数据库和Oracle自治数据仓库云便是最好的例证。”Andrew Mendelsohn如是说。

Oracle自治数据库（Oracle Autonomous Database），由Oracle数据库18c驱动，采用开创性的机器学习技术实现自治化，无需相关人力进行微调、修补、更新和维护，同时成本更低。今年3月甲骨文推出的Oracle自治数据库的首款服务——Oracle自治数据仓库云（Oracle Autonomous Data Warehouse Cloud）现已对全球客户开放，提供Oracle数据库的所有分析能力、安全功能和高可用性，即使在仓储工作负载和数据量发生变化时，也无任何复杂的配置、调优和管理。首创一步建仓模式，与企业现有本地部署数据库完全兼容，上云更便利；几分钟即可完成部署，以低廉的成本承担更多工作负载，运营更快速；独立的在线计算和存储拓展实现资源动态增减和按使用付费，资源配置更弹性。

自治技术本质上来源于当前不断涌现的商业及技术变革给企业带来的前所未有的挑战：

首先，外部小型初创竞争对手创新力强，借助全球云能力迅速扩展；其次，企业内部受制于复杂过时的IT环境，80%的IT人力禁锢在日常繁琐的工作，无暇顾及创新和IT现代化改造；第三，企业数据量平均每两年就会翻一番。数据存储、管理和分析难度急增；难以及时应对大量的网络攻击和数据泄露风险。同时，虽然现在IT领域的很多设备都采用了自动化设计，但仍然需要人的大量参与。以数据库为例，尽管很多操作都自动化了，但运行数据库还是需要数据库管理员付出很多时间精力。据IDC估计，高达75%的数据库管理成本是人力成本。而“自治”技术，是甲骨文洞察企业需求，全面助力企业面对这些挑战，并帮助企业数据库客户云转型的重磅武器。而不仅仅是Oracle数据库的简单升级。同时，“自治”技术又是甲骨文持续创新的传承，也许也是甲骨文继续保持领先的关键。

全栈——甲骨文核心优势的体现

不仅仅是在数据库领域，甲骨文立足企业上云需求，为客户打造业界最全面的企业云服务体系，是业内唯一一家提供本地和云端部署间真正实现工作负载无缝转移的云供应商。基于全球范围丰富的最佳实践经验，Oracle云稳操“三大核心优势”，赢得业界认同：

·      企业级——作为唯一一家始终专注于企业级市场的云厂商，甲骨文凭借可靠、安全、高性能的云能力，四十年企业服务基因和引领企业IT的创新实力，直击企业核心需求。

·      全栈式服务——甲骨文全面推出跨越SaaS、PaaS、IaaS三层的全栈式云服务，是全球唯一一家能提供全面、集成，且基于行业标准的全部三层云服务组件的厂商，让企业客户按需灵活选择上云时机，为企业上云全程保驾护航。

·      化繁为简——甲骨文在本地和云上数据库业务均采用相同的架构与技术，可无缝、一键式衔接，还推出了多种桌面和移动应用开发，及基于框架的无代码开发工具，避免“云迁移”带来的成本投入和安全风险，帮助客户快速轻松实现数据库“云升级”。

甲骨文公司副总裁及中国区云平台总经理吴承杨表示：“除了三大核心优势，作为云时代技术创新领航者，甲骨文紧紧把握时代脉搏，将人工智能、区块链、物联网等新兴技术真正落地于我们的三层云产品及解决方案，打造以‘红科技’为代表的全面技术产品，并与商业场景深入结合，全面支持开源开发，从IT端为企业业务发展注入新鲜血液和竞争力。”

甲骨文正在企业级AI服务方面引领潮流。甲骨文基于人工智能的数据业务平台，为企业提供贯穿数据完整生命周期的整合云平台，且可基于企业需求灵活部署在本地、公有云或是甲骨文独创的“本地化公有云”上，帮助企业轻松管理海量增长数据，将数据“变现”，从而更好掌握市场动向，提升经营效率，找到下一个业务增长引擎。

甲骨文刚刚正式推出的Oracle 区块链云服务（Oracle Blockchain Cloud Service）是“红科技”落地的又一实证，帮助企业更安全、高效地处理交易，并在全球供应链跟踪货物。Oracle 区块链云服务基于Linux基金会的Hyperledger Fabric的开源技术构建，创造性地为客户提供了一个开发平台来构建自己的网络，并快速集成 Oracle SaaS及其他现有的第三方应用，以及其他区块链网络和 Oracle PaaS 服务。此外，该服务还支持由用户提供区块链网络、连接其他组织以及部署和运行智能合约来更新和查询分类帐。

在中国，服务中国

自1989年进入中国，甲骨文始终坚持“在中国，服务中国”的初衷，以本地客户需求为创新动力。而在本次大会上的一个重磅消息——传闻已久的甲骨文中国数据中心将于8月份正式对外提供服务——则是甲骨文初衷的又一个有力证明。

据李翰璋介绍，甲骨文中国数据中心将通过腾讯云运营的数据中心对外提供服务，而且除了数据库云以外，中国数据中心将提供其他多种服务，包括中间件，Golden Gate云、Oracle Analytics Cloud（Oracle分析云）、移动云、管理云等。

李翰璋表示，甲骨文在中国已经有29年，员工数已经超过5000人。甲骨文的理念是给中国带来最先进的科技，并与全球是同步，这一点非常关键。甲骨文始终认为，必须实现这样的同步，才可以确保国外的客户到中国、或中国的客户到海外，都享受同等的科技，这在云时代是至关重要的。

李翰璋表示，甲骨文在中国的整体的业务，可以从三方面阐述。首先，甲骨文在中国的业务情况是优秀的。其次，甲骨文在中国的持续投入，从商业的层面来说也是与业务相辅相成的。第三，除了传统的业务以外，过去两年甲骨文的一体机、一体机的本地化以及由腾讯云运营的数据中心的落地等，再加上SaaS、PaaS跟IaaS全面的发力和超过一千多新员工的加入，这些都可以充分体现出及甲骨文对中国的业务和中国市场是充满希望和信心的。

李翰璋最后表示：“甲骨文中国对云非常有决心，也有信心。我们一直专注于服务企业级客户，以客户需求为中心全方位发展我们的云服务，奠定了甲骨文中国明确的云战略和独特优势。立足本地，放眼未来，甲骨文将持续深耕中国市场，履行承诺，集结各领域客户及合作伙伴同谋发展，共建生态，迎接自治未来。”
