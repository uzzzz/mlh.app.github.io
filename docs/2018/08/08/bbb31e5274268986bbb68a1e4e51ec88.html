商业银行开启科技金融银行新篇章：新时代，新金融，新银行

  • 柚子社区

数字化转型是目前各行业企业面临的首要难点，为充分展示各行业在数字化转型中对云计算的不同需求与特有的转型经历，BoCloud博云推出了【数字化背后的云引擎】系列文章，涉及互联网金融、银行、政务、能源、制造等行业，立足行业场景与业务痛点，讲述创新云技术如何推动企业数字化转型和业务创新。

一、新时代，新金融，新银行

自2014年开始，新经济概念开始在中国得到重视，随着互联网、云计算、大数据等新兴技术的发展，以信息产业为主导的新经济正在席卷全球。在此环境下，新经济的新特征对传统金融的产品、服务、模式等提出了新的要求，必须打造新金融模式来适应新经济的发展，必这其中新金融体系和新金融服务模式是新金融的两大重点。银行作为金融行业的支柱产业，在互联网商业应用对传统银行的冲击下，转型已迫在眉睫。

本期将为大家介绍，国内某大型股份制商业银行携手BoCloud博云，为应对银行互联网化发展趋势中，业务系统呈现出开放性、用户量不可预测、应用新功能更新频繁的新特点，如何实现构建基于分布式架构为核心的业务系统的数字化转型之路。

二、云转型战略的应对与挑战

该行作为国内第一家主要由民营企业发起设立的大型全国性股份制商业银行，在业界创造了多个第一，根据2017年美国《财富》杂志公布的世界500强名单，该行位居251位，英国《银行家》公布的全球1000家银行中，该行位居29位。

在科技引领金融服务转型的浪潮，该行一直深耕科技创新，科技金融银行建设中系统架构是核心基础。随着社会信息化发展，该行金融服务呈现出海量客户请求激增、数据处理运算繁多、个性化定制化产品增多、业务快速敏捷响应等新特点，而传统集中式的 IOE 架构已无法适应科技金融银行的发展需要：

  1. 大部分应用系统有开发、测试、准发布和生产4个部署环境，各部署环境不一致，导致代码从开发到上线环节多、部署复杂、容易出错，应用的快速部署开通受到极大制约，无法满足银行业务快速上线的要求；

  2. 银行应用系统部署以虚拟机为单位构建，系统的扩容经历虚拟机分配、软件安装、应用部署和测试、切割入网等环境，在业务量突增情况下无法应对进行快速扩展，系统缩容不能按需进行，导致系统弹性扩展能力不足，使资源存在一定的预留和浪费；

  3. 资源池的CPU和内存利用率低，致使大部分物理资源处于空闲状态，现有资源利用率低；

  4. 以虚拟机为基础的资源池化，并没有改变竖井化的建设模式，应用与平台之间并没有解耦，监控运维无法标准化，应用系统仍旧“烟囱式”建设。

三、云转型，变则通，通则久

单纯地依赖提高服务器运算性能或部署更多服务器来解决传统 IOE 架构问题，一方面会带来显著的成本增加，另外一方面也无法满足业务对时效性的要求。

作为中国市场化程度最高的商业银行，该行凭借对市场行业变革趋势的敏锐判断，深刻认识到金融行业转型的应对和挑战，不再是转与不转的问题，而是必须转的问题。互联网金融倒逼银行转型，主要就是表现在业务转型上，业务倒逼科技实现转型，而以实现敏捷业务为著称的云计算技术就是转型的必经路径。

基于以上原因，为进一步完善金融互联网化基础平台的产品体系，降低成本，实现应用的快速迭代和弹性，同时也进行“去 IOE ”的实践，经过一系列严格的需求调研与POC测试流程，该行选择了在 DevOps 解决方案上具有丰富落地经验的BoCloud博云，使用了标准X86服务器，构建基于轻量级虚拟化 Docker 技术的 DevOps 分布式核心平台。

BoCloud博云为该行提供的 DevOps 分布式核心平台，主要用途是为金融互联网创新应用提供平台支撑，主要目标建设分为：

  • 建设安全可靠、支持弹性伸缩、灵活拓展的基础环境，采用轻量级虚拟化技术Docker，对不满足该行需求的 Docker 进行改造和完善；

  • 建设更能支持海量客户、快速响应客户需求、易扩展、服务化的分布式互联网架构平台；

  • 整合自动化、可视化工具，满足基于 Docker的应用部署、运行和维护的易操作需求，实现自动化运维，保障银行业务系统的高可靠运行；

  • 对大型数据中心的IT资源进行统一监控管理，保证服务质量。

博云从以上方面分别为银行业务系统互联网化提供技术基础、自动和可视化管理、可靠性保障等功能。

四、云技术重塑科技引领动能

如何适应新时代趋势、挑战行业竞争，重新发挥科技对银行的驱动作用，已经成为银行业面临的重要课题。云技术正是帮助金融科技银行实现灵活高效业务流程和智慧化产品服务的重要技术手段。2018年初该行 DevOps分布式系统正式上线，实现了持续集成、自动部署、弹性伸缩、灰度升级、监控告警等完整的 DevOps功能，该行科技支撑业务能力进一步大幅度提升：

  • 核心业务项目研发迭代周期从数个月加快到1-2周更新一次，能做到快速响应业务需求；系统快速弹性伸缩能力可承担十亿级海量客户规模、以及促销活动等高并发交易量的需求场景；

  • 利用数据库中间件、分布式缓存、服务注册、消息中间件等技术建设的布式互联网架构平台，实现了分布式、高并发、高弹性的应用架构支持，有效提升系统响应速度和吞吐量；

  • 将直销银行等应用由传统的 IOE 架构改造成分布式系统架构，性能较原来提升6倍，解决了应用本身的容量瓶颈，硬件成本减少85%，分布式核心系统每年节约近90%维护成本；

  • 自动化运维体系支撑更复杂的服务层次、调用关系及系统状态，实现运维管理集中化、运维操作自动化、应用平台可视化、服务监控管理职能化，确保持续交付能力。

未来该行将继续围绕科技金融银行战略规划，持续提升技术能力、服务能力和创新能力，不断提升其数字化水平，打造行业标杆银行。
