Token Design随笔（七）：为什么Token会涨？（续集）

  • 柚子社区

上次我们说到，按照公式

P = T / (M * V)

可以得出结论：

如果总的交易量T大了，P就会涨

如果流通的Token数量M变少了，P就会涨

如果Token流转速度下降了，P就会涨

看起来挺对的，可没法解答我们上次提出的问题：

在Token使用场景中，需求方购买Token给市场造成的升值压力，和供给方拿到Token后抛售给市场造成的贬值压力，不是两相抵消了么？

流通速度极快（V极大）时，Token价格应该为0？

流通速度极小（V极小）时，Token价格应该为无限大？

总觉得不对啊。

————————

如果我们考虑Token的使用场景和使用价值，这时Token可能的使用方式有两种，一种是直接对应『具体供给 x 具体数量』，例如一个Token对应一个星巴克大杯拿铁，一个日本民宿1/1000的产权，或者1G * 1个月的存储空间。另一种则是仅仅作为计价单位，例如要约宋慧乔吃饭要10000个Token，要买一个加密袋鼠需要237个Token。

如果是第一种，我们会很悲惨的发现，这个Token的价格P，其实完全等同于它对应的供给的法币价值。T啊、M啊、V啊，是经济运行的结果，而不是我们可以操作这个经济的抓手，不可能通过操作这几个参数来影响P。

假设一个经济里，一个Token对应一个星巴克拿铁，你不可能去操作其他参数，让这个Token莫名价值1000块。价值1000时，没有人会用它换拿铁，交易T和流通速度都会降到0，整个经济也就崩溃了，理论上来讲用1000块钱来收Token的人得赔死，然后价格会回落到30多块。

费雪公式对应的是国家经济，国家内民众在货币选择上其实很被动，货币竞争压力小（一般意义上通胀一下，大家也不会直接放弃本国货币换成用外币或者黄金）。但私有经济体系则不是这样，有各种竞争替换品存在，如果你的经济机制不make sense是没人和你玩的。这是不能硬套费雪公式的原因。

在这种情况下，唯一的升值逻辑，就是对应供给价值的上升。比如这个Token是星巴克发的，星巴克生意越来越好涨价了，对应Token就涨了。这其实和Token Economy没半毛钱关系。

假如是第二种，用Token来计价，倒是说得通，以前一个鸡蛋要两张粮票，现在粮票升值了，两个鸡蛋才要一张粮票，涨了四倍。

可是，可是，这种场景下，粮票价值（价格）究竟是和什么相关呢？如果不仅能买鸡蛋还能买鸭蛋了，粮票价格会涨？还是说鸡蛋变多粮票变少才会涨呢？

如果是后者的话，价格其实还是收到V的影响，而不是T的影响。仅仅是因为T（使用场景）的增加，可能会导致大家手头留存的Token数量上升（反正能花掉），降低了整体的V，这才使得价格可能上升。

如果这样的话，干嘛这么麻烦花那么大精力设计使用场景和使用价值呢？

难道设计目标不应该是『持有价值』才对么？
