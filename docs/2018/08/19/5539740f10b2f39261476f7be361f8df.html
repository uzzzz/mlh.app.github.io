初链-解读初链白皮书

  • 柚子社区

        作为一个区块链行业内的技术，在偶然机会了解到了初链（TrueChain），决定将近期内的一些收获和了解以文章的形式记录下来，所以有了此文。

1.初步了解初链

        初链于2018年1月16日发布货币TrueChain/TRUE，经过三个月之后开始猛涨，最高价达到23.10，至文章发表时仍然拥有着不错的市值和优秀的换手率，综合分析下来现在依然是很有潜力的币种。

2.解读白皮书

       可以说现在的币圈、链圈人士都会很关心一个项目是否有白皮书，足已说明白皮书的重要性了，所以在大致了解TrueChain之后，我的第一步就是下载一份白皮书细细解读。凸显不出TrueChain特点的地方我就不再赘述了，文章里着重说一些TrueChain比较重要的内容。

（1）安全性

       首先要看的就是安全性的介绍，相信大家都会听说过很著名的门头沟事件、parity事件、USDT事件等等，这些事无时无刻不在提醒着我们要重视安全，要慎重选择平台。我只在白皮书上看到他们提到以POW(工作量证明机制)为基础，选举产生PBFT(拜占庭容错)节点，并且保证实时监控，如果节点出现问题时及时进行重新选举，却没有从白皮书上获取到多少关于用户安全的信息。从交易所的角度来看，我没有看到TrueChain如果被恶意操控市场价格、黑客盗取等事件导致的损失追回或补偿的相关立场，从币的本身上看，我没有看到货币存储、收发时会不会出现丢币的风险。所以这个模块被官方命名为“安全性”，我倒是认为叫“稳定性”更好一些。

（2）技术架构

       从技术架构上来看分为四层：一是底层服务；二是核心层；三是 API 接口层；四是智能合约层。具体设计在黄皮书和 github 的开源代码中有所体现，白皮书中介绍的最有特点的就是fPoW的挖矿机制，采用了水果链（FruitChain）的设计。

       这也是我第一次听说到“水果链（FruitChain）”的这个名词，在网上搜索了很多相关资料都没有介绍，甚至于fPoW的出生年月都是不详，但是我了解到TrueChain是首个真正实现fPoW的项目。要说到fPoW的特点就是解决了PoW的私挖矿，联合挖矿和不稳定的交易费等问题，可谓是一次勇敢的创新。如果是TrueChain刚刚开始时我写的这篇文章，我还会保持怀疑态度，但是现在我认为这次尝试是成功的。

（3）团队阵容

       有的时候不光要看做出来的事更要看做事的人，因为一个团队将会决定项目和技术的天花板有多高。

       从TrueChain白皮书的tuan团队介绍中可以看到，数位主要人物的介绍，都是清一色的名牌大学，不乏大厂出身之人。相当亮眼的就是作为CEO的张剑南和工程技术负责人Archit Sharma。团队阵容堪称豪华，而且他们也在TrueChain上证明了自己的实力。
