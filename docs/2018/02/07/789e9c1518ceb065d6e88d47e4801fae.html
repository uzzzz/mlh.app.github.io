身份认证一直都是一个非常神奇的存在，卡片身份证的时候我们认为芯片身份证很好，但是物联网时代，我们希望能把一些智能穿戴产品变成我们的身份认证，简单方便。但不管如何，这些数据的可能被复制的命运是不能更改的。

而储存我们个人信息的机构、单位，每一个都是高度中心化的单位，相互之间产生数据共享几乎是不可能的，这也就导致了有些人认为A是好人，有些人认为A是坏人。有些人则需要A证明A是他自己。

如何解决这种需要自己证明是自己的问题呢？耳朵找到了专门做ID领域区块链创业的IDchain团队进行了专访，探究为什么我们可以不用证明我是我了。

IDchain的创始团队首先聊了一下他们在生活接触到的一些痛点：比如多个银行 U 盾其实指向同一个人；传统的出生证明、房产证明、学位证明、食物安全检验标识和身份证等信息则更是由政府等专有第三方开具，它们相比 U 盾、短信验证码的成本更高。而认证行为是现代生活的基本元素，我们需要佐证这就是我，我的出生、学历、驾驶技能等等这些事情发生了即不需更改的重要属性。

什么是IDchain？

ID Chain（IDC 公鉴链）借助在加密算法的约束下，将各种用户身份信息等重要数据加密记录在一个个区块链上，这些记录不可篡改，100%生成即不可逆的这样一个ID Chain平台上，简单的授权与扫描即完成可快速完成认证。在ID Chain的平台中，我们通过去中心化存储和处理各种用户的属性和数据。

我们生活的环境其实一直都是一个高度中心化的环境，以每个银行为例，每个银行的存储的个人信息都是高度集中的，最近几年其实也是一直在做减法。但是流通信息上来说确实没有什么进展，就以我们办信用卡来说，每个人都办过信用卡，但是如果在一个新的银行办一个新的信用卡的话，你的卡片额度还是需要你从头开始，并没有一个统一的评判标准。

对于个人的风险程度，出了银行系统，外部的一些金融机构很难对一些高风险人物有一个非常系统的评估。不过也有很多机构对此进行努力，就像是我们的芝麻信用分，腾讯信用一样，有一个具体的数值，可以让陌生人之间有一个大概的评判标准。

一个ID跨界应用

ID Chain目前已经与香港以及大陆的运营商，银行业接洽，充分利用其现有的大数据，进行热启动，简单来说运营商的数据，基本上可以塑造一个很完整的用户画像，你有什么喜好，什么职业，月收入多少这些基本信息通过运营商的大数据都是可以非常完整的分析出来的。

为什么叫热启动，因为我们如果在他们的平台上上传信息的时候会发现，平台之上已经有了一些我们的基础信息，我们或者机构只需要把我们的信息补录的更加完整了。

一个ID跨界应用的例子，其实在原来是不敢想象的，就像是现在我们登陆各种网站的合作平台登陆一样，将来在任何网站也许你都不用注册，只需要输入自己的姓名，身份ID。最后输入一个平台的验证（类似谷歌验证码，10秒钟换一次，每次都是6位数），你就可以直接在平台登录，而且你不用再从新录入你的手机号码、邮箱账号，实名认证等等，因为你的ID就包括了你所有信息的，不用在进行繁琐的输入。如果你想要有多个账号的话可能性同样不会很高了。

小国家转型的契机

对于这样的技术在国内来看可能并没有什么特别之处，毕竟我们有身份证，每个人生来就已经有了一个独立的ID，虽然不能在各行各业通用，但是至少我们有比较完整的身份统计系统。

但是在一些小的国家，没有身份证，没有身份录入系统，而护照是最容易造假的身份凭证，一些小的国家的护照甚至都不设防伪标识。这会导致人员统计困难，假身份肆意横行。而目前出现的区块链技术，成为了很多小国家的救命稻草，通过区块链的分布式录入，把每个人的身份唯一化，即使动乱、户籍管理系统被摧毁，每个人的ID信息依然是存在的。

ID Chain举了一个最实际的例子，菲律宾已经计划在5年内建立起他们国内的身份证系统，并在2018年投入数千万美元进行搭建。

总结

IDChain采用了区块链技术来做认证是区块链技术的非常典型的应用场景，其核心是建立一个以技术为约束的信赖关系模型来解决点对点的信任问题，其透明化，去中心化，不可篡改，智能合约等优良基因在认证处理中完全发挥出来。

区块链技术最让耳朵君心动的就是可标记性，对于所有人的可标记性，这样可以让所有人为他的行为付出代价，在其他项目中，每个人的ID都是匿名的，而IDChain把匿名的ID变成了实名制，这样也就会让大家加对于自己的言行负责，不会出现网上的键盘侠，胡乱煽动群众的现象。

这样的技术目前确实还不能实现，而且对于这种ID技术的认证来说，地方、政府、机构多方面的支持也是有很长的路要走的。以后当你通过交易发现有一个人的各种信息有非常大的变化的时候，恭喜你，你可能发现了一个特情工作人员。
