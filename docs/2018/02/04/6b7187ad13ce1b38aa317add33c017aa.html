win10删除账户文件夹(C:\Users\***)后，无法登录账户的解决方法

  • 柚子社区

前两天要做点东西，需要用到Administrator账户的权限，因为懒得一直点以管理员运行，就直接启用了Administrator账户，PS:不要轻易尝试这个账户，毕竟权限太高。

启用和关闭的方式为：以管理员权限运行CMD命令提示符程序，输入命令net user administrator /active:yes即可启用，关闭使用，将yes替换为no即可关闭。

言归正传：在我用完Administrator账户后，回到自己的账户文件夹，脑子一热，把C:\Users\Administrator给删了，也就是直接把Administrator的配置文件给删了，再试着启用Administrator账户时，问题就出现了，第一次启动时是一直黑屏，强制关机重启后，经过几分钟的配置，会提示无法登录到Administrator，也就是Administrator是不能用了，虽然自己的账户还是能用，不过不能使用Administrator还是有些不放心。

各种折腾啊，甚至在虚拟机里装了个WIN10，然后把Administrator配置文件夹复制出来，当然，经过种种艰辛，复制出来的文件夹还是不能用的，今天又瞎折腾，终于鼓捣出了解决办法：

右键计算机-属性-高级系统设置-用户配置文件：

点进设置后

可以看到有个默认配置文件，也只有它能够复制，选中默认配置文件后，点击复制，目录填写为C:\Users\Administrator，也就是把默认配置文件写入Administrator配置文件中，重启登录时，Administrator账户就像第一次启用时那样，经过几分钟配置就能正常使用了。

 

哎，这两天真的是能瞎折腾，先把电脑搞得Administrator账户不能正常使用，又把手机搞得开机引导文件出错，就快断网回家了还搞出这么一堆事来，也真是惊悚。
