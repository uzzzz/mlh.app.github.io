(EOS开发教程 二)在EOS 3.0上创建、查看、锁定、解锁钱包，以及钱包的状态

  • 柚子社区

注：EOS本地环境搭建和安装目录请看上一章


1）启动EOS服务器

$ cd ～/eos/build/programs/nodeos
$ cd ./nodeos

2）创建钱包

创建一个默认名字为“default”的钱包，并返回钱包密码，请妥善保存该密码，新版本会经常用到它：

$ cd ～/eos/build/programs/cleos
$ ./cleos wallet create
Creating wallet: default
Save password to use in the future to unlock this wallet.
Without password imported keys will not be retrievable.
"PW5JuBXoXJ8JHiCTXfXcYuJabjF9f9UNNqHJjqDVY7igVffe3pXub"

要是之前用这命令创建过default钱包，会出错，提示：

Error 3140001: Wallet already exists
Try to use different wallet name.
Error Details:
Wallet with name: 'default' already exists at /Users/用户名/Library/Application Support/eosio/nodeos/data/./default.wallet

可以删除后重新创建，创建的钱包会在以下目录存储：

/Users/用户名/Library/Application' 'Support/eosio/nodeos/data

把该目录下的default.wallet删除，即可重新创建。也可以创建别的名字的钱包，用以下命令即可：

./cleos wallet create -n 自定义名字

3）钱包状态

钱包有open、unlock、lock三种状态，刚执行完创建命令，是open + unlock状态，输入钱包查看命令：

./cleos wallet list
返回的结果如下，名字代表open， *代表unlock： 
Wallets:
[
  "default *",
  "test *"
]

一旦EOS服务器重启（先CRTL+C，再./nodeos，详情看上一章），所有钱包变为非open + lock状态，此时再用该命令，我们看到，这时返回的结果空空如也：

Wallets:
[]
这是最新版本为了安全而专门设计的机制。钱包此时无法使用，要想把钱包由 非open +  lock 变为 open +  unlock，需要解锁，可以用以下命令。中途会提示输入密码，输入创建钱包时返回的密码即可： 
./cleos wallet unlock -n 自定义的钱包名字
或者用 
./cleos wallet unlock -n 自定义的钱包名字 --password 自定义的钱包的密码
要是解锁default钱包，还可以更简单，可以用以下命令并中途输入密码： 
./cleos wallet unlock
或者用 
./cleos wallet unlock --password default钱包的密码

4）锁定钱包

为了安全，可以锁定钱包，可以用以下命令：

./cleos wallet lock -n 自定义的钱包名字
要是锁定default钱包，还可以更简单，可以用 以下命令： 
./cleos wallet lock

此时再输入钱包查看命令，结果如下，没有后面的*：

Wallets:
[
  "default",
  "test"
]
