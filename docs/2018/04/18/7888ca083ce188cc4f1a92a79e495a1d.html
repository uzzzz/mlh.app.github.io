暴走时评： 据业内媒体最新报道，科技巨头苹果公司最近为Apple Pay引入了支付请求API（Payment Request API），支持其颇受欢迎的Apple Pay支付系统，同时还配有瑞波（Ripple）的Interledger扩展。据悉，Payment Request API能够通过标准API支持跨不同浏览器的多种支付方式，从而帮助降低使用的复杂性，但苹果公司并未明确提到瑞波或Interledger技术。

作者：BHUSHAN AKOLKAR    翻译：Clover

根据业内最新报道显示，总部位于库比蒂诺的技术巨头苹果公司（Apple Inc.）引入支付请求API（Payment Request API），用于支持其颇受欢迎的Apple Pay支付系统，同时还配有瑞波（Ripple）的Interledger扩展。前不久，该公司宣布推出网页版Apple Pay交易，因此，可以在macOS和iOS 11.3系统上的专有浏览器Safari OS 11.1完成此API的集成。

这一消息的公布就表明苹果公司给出了一种可能性，那就是未来该公司可能会使用瑞波构建的Interledger技术。如今，苹果是全球最大的技术公司之一，产品种类繁多，包括iPhone、iPad、Apple Watch和MacBook。几年前，该公司推出了Apple Pay功能，允许通过互联网和PoS设备进行支付。然而，苹果公司面临的主要瓶颈之一就是缺少支持多种支付方式的方法，从而无法让更多商户接入。

Payment Request API就能够通过标准API支持跨不同浏览器的多种支付方式，从而帮助降低使用的复杂性。Interledger 支付方式是Payment Request API的拟议方法之一，商户能够使用这种支付方式请求支付，以便遵循Interledger标准。


什么是Interledger支付方式？

顾名思义，Interledger协议就是一种允许跨多个账本发送付款的开源协议。Github上的Interledger社区将其定义为一种“支付方法，例如，与支付请求API一起使用。有了它，商户可以请求符合Interledger标准的付款。”

Interledger协议包含许多可应用于任何支付方式的标准，如地址、双因素认证或签名。现有支付网络的一个主要问题是支付网络自身互不兼容。Interledger有助于解决这些问题，因为其开放式架构和框架允许包含不同值的支付系统彼此进行交互与通信。

Interledger还允许在不同数字资产之间进行支付，同时消除调解错误的风险。

瑞波就是这样一家初创企业，帮助在数字货币行业与金融机构之间搭起了一座桥梁，这些机构目前被认为是全球金融领域的主要竞争对手。与其争论不休，瑞波相信通过使用区块链技术提高现有支付系统的效率，可以与中央银行和其他金融系统更好的协调工作。

请注意，Interledger Payment Method的编辑之一Adrian Hope-Bailie属于Ripple。但引入支付请求API的苹果公司并未明确提到瑞波或Interledger技术。考虑到支付请求API和Interledger方法之间的关系，两者之间可能存在相互联系。

本文仅代表作者个人观点，不代表区块链铅笔的立场，不构成投资建议，内容仅供参考。
