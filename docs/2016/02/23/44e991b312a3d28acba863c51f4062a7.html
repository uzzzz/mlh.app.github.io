我遇到的问题

安装cmake,出现：

the path installer too long ,can’t add to ….
并不是你的环境变量内容太多了，而是你的计算机有点问题，或者其他问题。

解决办法:

手动添加，例子：path: …;C:/cmake/bin

典型错误：

CMake was unable to find a build program corresponding to “MinGW Makefiles”.
CMAKE_MAKE_PROGRAM is not set. You probably need to select a different build tool.

解决办法：stack_overflow这里很详细

我的办法很简单：添加环境mingw32：path:..;D:\Photo\Tools\mingw492_32\bin

遇到这个错误，非常讨厌，不过这个是很常见的错误



找到QT_QMAKE_EXECUTABLE，value 改为:/Qt/Qt5.3.1/5.3/mingw482_32/bin/qmake.exe
其他情况，依次类推。

最终 Congratuations!

实际上，这里可能还有问题：

你在编译的时候，可能只能编译到一般不到，就挂了。

原因分析：

1.你没有安装OPenGL，然而，你却选择了它编译。

2.版本不匹配，出现大量错误，…/huigui/…cpp error1, ……error2, 就是一些莫名奇妙的错误。就是cmake,opencv ,Qt版本的问题，实际上，这才是，所有问题的根源。安装Qt要选择含有mingw和openGL版本的，网上的教程，都是基于这样的基础的。

你在copy文件夹的时候，文件属性默认是只读的，所以，你会在编译的时候遇到，permission denied错误。

最后，献上大师的博客参考。码谜网
