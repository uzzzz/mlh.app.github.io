1、transparent_hugepage警告问题
在/etc/profile中写入如下脚本

if test -f /sys/kernel/mm/transparent_hugepage/enabled; then
echo never > /sys/kernel/mm/transparent_hugepage/enabled
fi
if test -f /sys/kernel/mm/transparent_hugepage/defrag; then
echo never > /sys/kernel/mm/transparent_hugepage/defrag
fi

2、cent7修改主机的hostname

hostnamectl set-hostname centos7

3、cent7操作防火墙命令

启动： systemctl start firewalld
关闭： systemctl stop firewalld
查看状态： systemctl status firewalld
开机禁用  ： systemctl disable firewalld
开机启用  ： systemctl enable firewalld

4、当通过集群中节点镜像做为虚拟机还原为新的集群节点时，dn数量不对的问题
原因：由于集群中的节点镜像已存在一起dn的旧数据，在/tmp/中的hdfs目录，或是在hdfs data dir中，导致dn的 live数量不对或是经常变化的问题。
解决方法：将还原出来的集群节点的/tmp/*和hdfs data dir目录的数据全部删除后，重启服务即可。
在当下集群中的操作方法是：

rm -rf /tmp/*
rm -rf /hadoop/hdfs/data/*
