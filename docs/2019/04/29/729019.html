译者：风车云马 

  

在国内,“谷歌云”这个词并不像“亚马逊AWS”、“微软Azure”、“阿里云”等热词一样，能给新闻网站带来可观的流量以致于稍有动作都会成为各大媒体的座上宾。但近日谷歌云发宣布的这件事你不得不关注一下。


在最近一次Cloud Next会议上，谷歌宣布了Anthos(企业混合多云平台)的通用性。


尽管在这次活动中有100多个平台发布，Anthos脱颖而出的原因很明显。它标志着谷歌正式进入企业数据中心，也是第一个来自主流公共云提供商的官方多云平台。


从Sundar Pichai到Thomas Kurian，再到Urs Holzle，Anthos被誉为下一代技术。很明显，谷歌的高层领导团队为Anthos感到骄傲。

 

Sundar Pichai, CEO  GOOGLE


Anthos与其他公共云服务不同。它不仅是一个产品，而且是多个服务的支柱品牌，这些服务与应用程序现代化、云迁移、混合云和多云管理等主题保持一致。


尽管谷歌Cloud Next的覆盖内容很广，也具有一般的可用性，但是Anthos的声明还是令人困惑。其说明性文档很少，而且服务没有完全集成到自助服务控制台。除了混合云连通和多云应用程序部署之外，我们对谷歌中的这项新技术知之甚少。


这里提供Anthos的一个整体介绍。


构建模块


Anthos的核心是采用了当今时代最流行的开源项目——Kubernetes。Anthos是建立在谷歌Kubernetes Engine (GKE)的坚实基础之上的，谷歌Kubernetes Engine是谷歌云平台上提供的托管容器服务。Anthos还有其他关键技术增强了Kubernetes的力量。


下面看看Anthos的核心构建模块：


1)谷歌Kubernetes Engine——Anthos的中央指挥控制中心。客户使用GKE控制平台管理运行在谷歌云、on-premise数据中心等云平台上的分布式基础设施。


2) GKE On-prem ——谷歌提供了一个与GKE一致的基于kubernetes的软件平台。客户可以将其部署到任何兼容的硬件上。从升级Kubernetes版本到应用最新补丁，谷歌将把它当作GKE的逻辑扩展。需要注意的是，GKE on -prem作为一个虚拟设备运行在VMware vSphere 6.5上。对其他管理程序(如Hyper-V和KVM)的支持正在进行中。


3) Istio——该技术支持跨平台的联合网络管理。Istio充当服务网格，连接数据中心、GCP和其他云部署的应用程序的各个组件。它与VMware NSX、Cisco ACI等软件定义的网络无缝集成，还包括谷歌自己的Andromeda。在投资使用F5等网络设备的用户可以将Istio与负载均衡器和防火墙集成在一起。

4) Velostrata——谷歌于2018年获得云迁移技术，为Kubernetes公司提供云迁移技术支持。Velostrata提供了两个重要的功能——on-prem物理/虚拟机，以在GCE实例中创建副本；以及将vm转换为Kubernetes应用程序(pod)。这是业界第一个由谷歌构建的physical-to-kubernetes (P2K)迁移工具。这个功能可以在Anthos迁移时使用，目前还处于beta测试阶段。


5) Anthos配置管理——Kubernetes是一个可扩展的策略驱动平台。由于Anthos的客户要处理运行在各种环境中的多个Kubernetes，因此谷歌通过Anthos简化配置管理。通过部署构件、配置设置、网络策略和密码，Anthos配置管理可以配置并应用于一个或多个集群。它可以看作是一个版本控制的、安全的以及与策略和配置相关的所有内容的中央存储库。


6) Stackdriver——对Anthos基础设施和应用程序进行观察。客户可以跟踪Anthos中运行的集群的状态以及部署在每个托管集群中的应用程序的运行状况。它是一个集中监视、日志记录、跟踪和可视化平台。


7) GCP云互连——没有企业数据中心和云基础设施之间的高速互连，任何混合云平台就没有意义。当数据中心与云连接时，GCP云互连可以提供高达100Gbps的速度。客户还可以使用Equinix、NTT Communications、软银等公司提供的电信网络，将数据中心扩展到GCP。


8) GCP市场——谷歌创建了一个可以在Kubernetes上运行的ISV和开源应用程序的列表。客户可以通过单击在Anthos部署Cassandra database和GitLab等应用程序。谷歌最后会提供一个由内部IT维护的私有应用程序目录。


谷歌的产品管理团队为Anthos服务做了大量的工作。


Greenfield 对比 Brownfield 


Anthos的核心是应用现代化程序。谷歌预测未来所有企业应用程序都运行在Kubernetes上。因此，它投资了Velostrata等技术并将vm升级为容器。


谷歌为VMware vRealize构建了一个插件，将现有的vm转换为Kubernetes Pods。即使是PostgreSQL和MySQL，也可以在Kubernetes中作为状态集迁移和部署。


依照谷歌典型的做法，该公司正在淡化on-prem vm向 cloud vm的迁移。传统业务线应用程序(如SAP、Oracle Financials和Peoplesoft)的客户可以继续在on-prem vm中运行它们，或者选择将它们迁移到计算引擎vm。Anthos在技术上可以给Kubernetes的vm和容器化应用程序之间提供互操作性。


谷歌希望在Kubernetes中使用所有微服务的应用程序(greenfield)，同时将现有的vm (brownfield)迁移到容器中。运行在非x86架构的程序和其他应用程序将继续在物理机或虚拟机中运行。


广泛的行业合作


谷歌经常被批评缺乏与其他行业参与者的合作。怀着打败微软和亚马逊的雄心，谷歌已经准备好与生态系统中成熟的行业参与者合作。这一举措无疑将有助于谷歌将其定位为企业玩家。


要想在Anthos取得成功，谷歌必须通过现有的企业参与者来推动。它正在寻找与微软(Microsoft)和亚马逊(Amazon)携手推出混合云的合作伙伴。


思科正在将HyperFlex、ACI、SD-WAN和Stealthwatch Cloud扩展到Anthos，交付经过验证的软件定义的网络功能。VMware正在为VeloCloud的Pivotal Kubernetes Service (PKS)和SD-WAN添加Anthos支持。它还将其流行的SDN NSX与运行在Anthos中的Istio集成在一起。


有12多个软体开发商支持Anthos的发布。从Citrix到NetApp再到MongoDB，有很多平台提供商将他们的软件与Anthos集成在一起。


Kubernetes


作为Kubernetes的创始人，谷歌对容器管理也略知一二。当Docker开始吸引开发人员时，谷歌意识到这是对外释放Kubernetes的最佳时机。该公司还迅速在公共云计算领域推出了业内首个托管的Kubernetes。尽管Kubernetes提供了十几种托管服务，GKE仍然是运行微服务的最佳平台。


凭借对Kubernetes的深刻理解和大量投资，谷歌希望在容器（containers）和微服务（microservices）的美丽新世界中占有一席之地。该公司希望企业从vm跨越到Kubernetes，以支持现代应用程序的运行。


Anthos是谷歌的一个大胆举措。它冒着很大风险摒弃老套的混合云，而其竞争对手正利用这一点来吸引企业。Anthos必然会与Microsoft Azure Stack和VMware的AWS进行比较。谷歌与其他产品的根本区别在于容器和Kubernetes。


谷歌希望利用其在Kubernetes和云原生生态系统中的优势地位。围绕Kubernetes快速建立企业战略。谷歌的目标是通过Anthos成为容器生态系统的VMware，将其管理程序和软件定义的基础设施推向企业。


但微软也在containers（容器）和Kubernetes上大举投资。它将旗舰版公共云服务引入私有云，从而模糊了Azure和Azure Stack之间的界限。微软可以用其Azure Kubernetes服务提供Anthos的服务。


如果谷歌一切都顺利的话，Anthos最终将成为企业运行的首选平台。


企业准备情况


让思科和VMware的高管与托马斯•库里安(Thomas Kurian)一起登台，是谷歌的一项声明，表明他们已经为企业做好了准备。这无疑增加了企业客户投资GCP和Anthos的信心。


Kubernetes被认为是一个高度技术化、网络化的平台，吸引着开发者和运营商。有了Anthos，谷歌将改变人们对Kubernetes的看法。它将平台变成了一个可行的、可靠的、企业级的混合云平台。


由Thomas Kurian掌舵，谷歌将目光投向了企业。如果谷歌充分利用伙伴关系，并继续与关键行业参与者合作，它将与Anthos一起撼动企业基础设施市场。


云原生生态的巨大机遇


Anthos增加了云原生生态系统的信心。正如前面提到的，谷歌正试图成为Kubernetes的VMware。


为了Anthos取得成功，谷歌必须依赖于社区和生态系统。这为新兴的初创企业提供了机会，它们可以提供完善云计算原生系统的利基产品。像Tigera, Portworx, Robin, Confluent, Cloudbees这样的初创公司将会从Anthos中获得巨大的收益。


服务提供商和系统集成商正在为Anthos做准备。从规模较小的本土企业，到埃森哲(Accenture)和Cognizant等全球性公司，谷歌的混合营销策略转化为数百万美元的服务机会。


随着谷歌向企业宣传和销售Kubernetes，许多新兴企业将乘势而上，开始推向企业。谷歌对Anthos的巨大押注将有利于行业、开源社区和云原生生态系统加速Kubernetes的采用。
