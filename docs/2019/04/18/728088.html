分享给转行的“产品应届生”：产品岗面试中你不得不知道的事

  • 柚子社区

导读：目前，产品经理岗位不再像之前一样门槛低，而是对各项能力的要求越来越高，想要顺利拿到offer，提前做好充分准备是非常必要的。

正文

首先简单介绍一下自己的工作情况，本人今年6月份毕业，大学的专业是物联网工程。我目前就职于一家网络安全公司，职位是产品助理。

之前未做过产品相关工作，但是出于对产品的兴趣（这么说谁信？！！）和不想做码农的原因，打算转行做产品。为方便阅读，正文文章将按以下思路展开叙述：

一、转行前准备

1. 心理上

首先，转行就意味着要承担一定的风险，切不可盲目跟风或者没有目的的转行。转行之前一定要在心理上做好充分的准备，才能对以后可能遇到的问题或者现实带来的落差有个心理准备。

我想既然你想转行做产品，就应该大致知道产品助理/经理（PM，以下一律用简称）是做什么的。通俗来讲，就和你在网上看到的描述差不多：不需要太懂技术（其实更确切的说是不用你做技术或敲代码），懂点设计，懂如何沟通就基本OK。但现在来看，懂“点”已经不行了，不过在大城市找个助理还是能找到的，前提是简历上有点料。

所以，你一定要搞清楚PM到底做什么的，然后评估自己对这种工作到底感不感兴趣，如果确实有兴趣（没能力也没关系），那就基本可以试着做产品了。毕竟刚毕业，找一份自己感兴趣的工作对以后的发展更重要。

2. 行动上

一旦确定要转行产品了，接下来一定要为面试好好做准备，因为我们经验不多，不准备肯定会被问住，况且未来在产品路上我们还有很多东西要学习。行动上的准备也可以简单粗暴的分为两种：花钱报班or自学。这个根据自己情况自行判断，报班肯定学的更系统，更扎实，重要的是有效率，自学也不是不可以，助理的门槛还是很低的。我因为经济状况没有报，所以没资格大谈报班的效果，只分享一下对自学方面的个人经验。

首先，把能搜集到的资源统统整来，比如关注PM相关公众号，买PM面试技巧有关的书籍，或看免费网课。让自己短时间内对产品有个大概了解，对面试技巧有个深入掌握（下文会提到一些面试可能会问到的问题），也检验一下自己通过一段时间的了解学习，是不是真的对产品感兴趣。上面提到的与产品有关的资源请自己找，现在互联网很发达。我只推荐一本书，个人认为这本书对我帮助很大：《产品经理面试攻略》，作者：鞠业旭。看完基本能了解产品是做什么的，而且能对做简历有个很好的认识与帮助，非常适合小白和找工作的人。

做以上工作的同时也可以问问周围同学有没有做产品的，或者学长学姐，找一个有经验的人直接沟通还是非常有帮助的。我当时就联系到一个学长，恰好大学还认识，给我传授了好多经验，可以说对我很有帮助了。

行动上的准备还得靠自己努力，一定要拿出个态度好好对待，只要努力，一定可以成功转行的，而且还算容易。

二、简历准备

简历的准备可以说非常重要了，因为现在找工作基本是先投简历再面试，简历不通过，面试准备的再好也没用。简历是要分行业情况以及具体面试职位有针对性的准备的。比如，你学的计算机，简历就基本定位互联网方面的PM了，而互联网又分为好多行业，娱乐，服务，游戏，金融等等。选择一个行业，然后对自己过往经历进行一个包装，也就是尽量让自己的经历向产品方面靠，具体怎样写可以参考我推荐的那本书。其次，还需要针对你的意向公司的具体职位的应聘要求来给简历加工润色。

简历并不是说做一份，然后海投就可以了，毕竟你做的不是Java工程师。产品包括各个行业，一定要有针对性的做，有针对性的投，才不至于被石沉大海。海投可能会有公司联系，但就不一定是你喜欢的公司或者岗位了。其他的准备工作就不说了，和你面试其他工作一样，该咋写咋写，可网上找简历制作技巧，最重要的还是对你的项目和实习模块做工作。

三、四次面试经历

首先，找几个招聘平台提前投简历，约面试时间。平台很多，Boss直聘，智联招聘，拉钩，51job等，各平台各有特点，可自行体验，选择适合自己的。我是在boss上找到的，boss处理信息还是很快的。我的目标城市是北京，因为大城市工作机会真的很多，像PM这种职位，去小城市基本没有，或者只招经验丰富的。

以下说一下自己的其中4次面试经历，这几次经历都是自己凭借当时印象写下的一些重要信息，具体细节没有，请自己参考一下就好。面试一般都挺轻松的，放松心态展示自己就好，不必放低姿态，面试就是双方互相选择的一个过程。

1. 某科技有限公司（物联网方向）

谈了50分钟左右，产品部人员面试问了好多专业知识，人事部主要确认能不能出差，未谈到工资问题。（后来才知道未谈到薪资问题基本就是没有招你的意向）

问题:

  1. 如果产品部和市场部出现分歧，你作为中间人怎么处理？

  2. 你怎么规划你的这些项目时间的？假如技术人员并未能按照规定时间完成项目，你怎么看待和处理这件事？

  3. 以后面对的需求人员是客户，你怎样和客户沟通需求问题？

  4. 产品开发流程你大概讲解一下？

  5. 你为什么打算进入产品这个行业？

  6. 有没有简历以外想让我了解的？最好谈一下与产品相关的经历，或者你其他优势？

  7. 你能说一下对我们公司业务的看法吗，比如你能说一下对我们我们公司“新零售”的看法？

2. 某国际教育公司

总体面试非常轻松，夸我简历制作精致，材料准备细心(我主动提供了自己的大学成绩单和所有证书复印件)。

整个过程基本都是小姑娘一个人和我介绍这个职位的在本公司的工作内容以及公司业务的一些介绍，没有问我技术方面的问题，说入职后有一周的带薪培训，谈到了工资问题，月薪5.5k加绩效和五险，朝九晚六有双休，根据自己能力有两个等级的晋升空间。说本周五下午4点让项目经理给我复试，深入了解我一下。

感觉很稳，但是职位是项目助理不是产品助理，后来放弃了复试，这里想说的是产品助理和项目助理虽然做的工作有相似性，但是最好不要拿产品的简历去面试项目助理，人家会认为你对自己的职业规划定位不清。

3. 某办公软件公司

比较重视的一家公司，面试过程比前两个更专业点，先是50道逻辑题的笔试，我答对了37道。紧接着是一个姐姐给面试，估计是产品部的，问了好多问题，围绕我简历也说了好多。谈我的生活、经历，尽量去了解我，问的问题很深，不断顺着问题的根深入问，大概谈了近1个小时。

总体不错，面试前一天了解了他们的产品，对今天面试很有帮助，最后给我留了一个作业，5天后交给她，算是最后的一个考验。 （可惜我没做，原因是我觉得有欺骗方案的可能，因为之前认真交过一次作业最后却没信了，很生气。面试完留作业的情况请自己斟酌，因为我也没啥经验）

问题（虽然问的少，但是问的深）:

  1. 优缺点？

  2. 生活或经历中认为最困难的一件事？

  3. 为什么转型产品？

  4. 你觉得产品经理需要什么品质？

4. 某IT公司

谈了大概30分钟，先是产品部leader,后是人事部。除了问的一个技术问题忘记外，没什么尴尬的事。产品那个人好像对我专业有意见，想招信息管理或网络专业的学生，最晚下周二给电话通知（3天左右）。

问题:

  1. 为什么这么晚才找工作，之前干什么了？

  2. 为什么转行产品，你对产品岗位的理解？

  3. 对我们公司有了解吗？

  4. 如果说要达到入职职责的要求，你觉得需要多久

总结

现在PM已经不像以前那样稀缺了，门槛也不像以前一样低了。

现在转行产品基本功需要扎实些

要找到一份产品工作，需要机会+能力（软硬）+热情+运气。所以现在趁着年轻，还有选择的资本，尽量选择自己感兴趣的行业，先踏实干，让公司看到你的态度与真诚。先写这么多，有补充的想起来再更新。
