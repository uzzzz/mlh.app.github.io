微软 GitHub 旗帜鲜明抵制 996！

  • 柚子社区

作者 | 伍杏玲

出品 | CSDN（ID：CSDNnews）

自3月27日996.ICU话题诞生以来，已引发国内外持续不断地关注和热议。国内大佬忙着发声，主流浏览器忙着屏蔽项目的GitHub地址。

而马云几天前谈的“996成功论”，被图灵得主Yoshua Bengio直斥：“现代奴隶制”。

昨天，IT界老大哥微软联合程序员的“港湾”GitHub为中国程序员发声：我们挺996.ICU！并大拍胸口保证：GitHub是我的，你们中国程序员尽管折腾，我们会保住996.ICU项目不会被删库跑路的！


微软和 GitHub 官宣：我们挺 996.ICU！


昨天，微软和GitHub的工程师联合写了一封信，宣布支持996.ICU运动。微软和GitHub的员工了解到中国程序员正陷入“水深火热”中：“中国的技术从业者正在利用 GitHub 吸引社会各界对中国科技行业中许多公司非法用工行为的关注。”

在信中介绍道“该 GitHub 项目被命名为 996.ICU，意为‘如果按照 996 的模式工作，那（你）以后就得进 ICU 了’。该项目旨在呼吁中国的科技公司遵守中国劳动法和国际劳工公约。”

他们明白996.ICU在中国面临的处境很艰难：“自从这一话题爆发以来，腾讯和阿里巴巴等中国国内的浏览器已经开始限制用户访问 GitHub 上的 996.ICU 存储库，并警告用户说该存储库中包含非法或恶意内容。”

所以坚定地表示支持996.ICU，为什么呢？

他们说：“这是一个跨越国界的问题。同样的问题也渗透到微软和整个行业的全职和临时工作中。我们必须与中国的技术从业者站在一个立场上的另一个原因是，历史告诉我们，跨国公司会把工作外包出去，利用不健全的劳动标准谋求利润最大化，迫使工人在竞次的过程中互相竞争。”

所以“我们必须跨越国界，万众一心，以确保世界上的每一个人都能够获得公正的工作条件。”

最后微软工程师承诺会：“在任何情况下都不要删改 996.ICU 的存储库，并保持其对所有人都可见。我们将尽我们最大努力继续支持 996.ICU 运动，并有效利用 GitHub 平台为中国科技工作者发声，传播他们的故事。”

截至发稿，目前已有100位微软工程师签名支持。

很快，国内程序员们在Issues纷纷对微软的“点赞”： 

   

在未来，我们的后代在撰写IT编年史里会不会留出一个章节，来讲述这震惊中外的“程序员996.ICU起义”呢？

为什么在国内知名浏览器屏蔽996.ICU，国内大佬大谈“996是一种福气”、“996是奋斗”时，反而会得到国外大佬和工程师的支持呢？

因为996的前方可能是“奋斗”、“成功”、“超越”时，但其背后皆是牺牲。

996的背后，皆是牺牲


一天24小时，一周7天，对每个人是一样的。倘若时间都在“996”、“8116+8”的话，留给健康、留给家人、留给生活的时间又剩多少呢？

马云问，一个人如果不付出超越别人的努力和时间，怎么能够实现想要的成功？他同时表示阿里之所以有今天，正是靠自己996拼搏而成的。

那背后默默无闻的妻子张瑛是否也这样认为呢？

阿里刚创业时，马云与张瑛携手创业，马云是01号员工，张瑛是02号。随着阿里的壮大，两人一心扑在工作上，疏于管教孩子。等发现时，孩子已沉迷于游戏，并不听他们的管教。

夫妻俩后悔不已，马云跟张瑛说：“你辞职吧，我们家现在比阿里巴巴更需要你，你离开阿里巴巴，少的只是一份薪水，可你不回家，儿子将来变坏了，多少钱都拉不回来。”

当时作为阿里巴巴中国事业部总经理的张瑛为了家庭，只好辞职回家，彻底地退到马云背后，当个“贤内助”，鲜少出现在公众面前。阿里巴巴上市时，股东名单也没有张瑛。

倘若没有张瑛为家庭牺牲事业，马云何以言说“成功”呢？

当看到刘强东“8116+8”工作制时，笔者不禁汗颜：工作日从早8点到晚11点，一周工作六天，周天工作8小时。笔者掐指一算，如果按照周天下午5点下班，孩子晚上10点入睡的话，刘强东留给孩子的时间仅剩周天的5个小时，一年52周，5*52 = 260，再除以24小时的话，东哥一年满打满算仅陪孩子11天。

刘强东需要“地板闹钟的故事”，可孩子需要睡前故事。

任正非曾接受访问说，自己对孩子们亏欠良多。他青年参军，一年休假回家次数少；中年创业，每天工作16个小时以上，吃住吃住都在办公室，从来没有节假日和周末的说法。所以他和孩子的关系并不亲近。

任正非是位成功的商人，却是个不合格的父亲。后来他也后悔没有陪伴孩子的童年，没有参与孩子的青少年时代。

当我们谈“996.ICU”项目时，不光仅谈前面的“996”，还有背后的“.”和“ICU”。毕竟996的背后，皆是牺牲。

现在能高谈论阔“996“、“成功”、“奋斗”的仅是目前已成功的还活着的大佬们，有谁曾问过那些为工作倒下，或艰苦创业失败的家属们，如果一切重来，还会让他累死累活地996工作吗？


如何解开“996.ICU”的矛盾呢？


虽然目前众多大佬为996.ICU发声，还有此次微软和GitHub的联合“上书”，可经历近一个月后，程序员该加班的加班，该996的996，假如能有机会改变，我们又该如何解开“996.ICU”的矛盾呢？

可能正如当当的李国庆说的：

第一，不用上升到剥削的道德批判。

劳资双方是谈判和双向选择出来的。互联网时代信息高度透明，个人选择企业前已经充分了解了，而企业面试时经常询问员工可否加班。

第二，他虽是坚决反对996，但他说“我也反对一周5x8=40小时工作。我主张50到54小时工作，但弹性，且结果导向。”

亦或者如日本微软的“一周三休”？

据日媒报道，日本微软将在今明两年的8月进行“一周三休”的政策。全体员工将在8月的每个周五享受“特殊带薪休假”，所有办公室停工，公司还补贴员工生活、旅游、学习，高达十万日元。

社长平野拓说：“希望全体员工能在更短的时间内，好好工作、好好休息、好好学习。改为一周工作四天后，按照旧的工作方法可能无法达到原有的产出量，这也是引导员工思考新的工作方式、思考方式。”
