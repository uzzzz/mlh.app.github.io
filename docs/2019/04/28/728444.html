近几年，物联网成为各行各业和资本争相追逐的风口，而作为物联网产业生态中的关键组成部分，物联网平台也成为科技巨头们战略布局中重要的一环。就国内而言，百度有“天工”智能物联网平台、阿里推出了 Link 物联网平台、腾讯发布了 QQ 物联智能硬件开放平台......

从国外市场来看，亚马逊有 AWS IoT、微软有 Azure IoT、Google 于去年推出了 Android Things 物联网平台，而在近日，Mozilla物联网团队也为物联网市场发布了自己研发的平台——Mozilla WebThings。

译者 | 弯月

责编 | 屠敏

以下为译文：

近日，Mozilla物联网团队很高兴地宣布，在开发人员和制造商社区的热切盼望中，经过两年的开发和七个季度的软件更新之后，Project Things终于结束了实验阶段，并迎来了一个新的名字 —— Mozilla WebThings。

Mozilla的使命是：“确保互联网是一个全球性的公共资源，开放且人人可用。追求一个真正以人为本的互联网，每个人都被充分赋权，安全而独立，能塑造出个性的体验。”

Mozilla物联网团队的使命是创建一个体现这些价值观的Web服务实现，并帮助推动物联网标准的安全性、隐私性和互操作性。

Mozilla WebThings是一个通过Web监控设备的开放平台，主要包含两个核心组件：

  • WebThings Gateway ：一个专注于隐私、安全性和互操作性的智能家居网关。

  • WebThings Framework：一组可重用的软件组件，可以帮助开发人员构建自己的Web组件。

我们期待将来Mozilla WebThings软件可以安装在商业产品上，通过这些产品为消费者提供值得信赖的“智能”联网家庭的代理。


WebThings Gateway 0.8


WebThings Gateway 0.8版本已经可以开始下载了。如果你有Things Gateway，那么它应该能够自动更新。这个最新的版本包含一些新的功能，你可以私密地记录所有智能家居设备的数据，还有新的警报功能和新的网络设置用户界面。

日志

你是不是想知道外出期间，你家大门的开关次数？你是否感到好奇插入到智能插头的电器的能耗？在新的日志功能下，你可以私密地记录所有智能家居设备中的数据，并通过交互式图形将这些数据用可视化的方式显示出来。

如果你想启用新的日志记录功能，那么可以点击main menu（主菜单） -> Settings（设置） -> Experiments（实验），并激活“Logs” （日志）选项。

然后，你就可以在主菜单中看到“日志”选项了。你可以点击“+”按钮，选择要记录的设备属性，包括数据的保存时长。

你可以通过该时间序列图，按小时、天或周查看以前的数据。虽然该功能仍处于实验阶段，但是查看这些日志可以帮助你了解智能家居设备正在收集的数据类型，并有助于你思考是否愿意通过第三方服务与其他人共享数据。

注意：如果从Raspberry Pi上的SD卡启动WebThings Gateway，那么请小心会有大量的数据记录到SD卡中，这可能加速SD卡的损耗！

警报

家庭安全和保障是智能家居系统的巨大潜在优势。你在上班的时候，如何才能知道家里是否响起了警报？如果附近有人听到了，他们会采取行动吗？他们知道该打电话给谁吗？WebThings Gateway 0.8为烟雾报警器、一氧化碳报警器或防盗报警器等设备提供了新的报警功能。

这意味着现在你可以检查警报是否处于激活状态，并设置规则，在你离家期间如果有警报被触发就通知你。

网络设置

在以前的版本中，如果你想要把网关从一个无线网络迁移到另一个无线网络，而迁移前的Wi-Fi热点依然处于活动状态，那么该迁移操作就只能通过Raspberry Pi的控制台上通过命令行进行。然而，在0.8版本中，你可以通过Web界面重新配置网关的网络设置。你可以在Settings（设置） -> Network（网络）中找到这些设置。

你可以配置以太网端口（使用动态或静态IP地址），或者重新扫描可用的无线网络，并更改网关所连接的Wi-Fi接入点。


WebThings Gateway的无线路由器


我们很高兴地宣布，我们一直致力于基于OpenWrt的新WebThings Gateway的构建，该网关的目标是消费级无线路由器。该版本的WebThings Gateway本身可以充当wifi热点，而不仅仅是作为客户端连接到现有的无线网络。

这是我们的网关软件发展的新阶段，因为我们正在朝着消费级无线路由器的方向推进软件的发展。进一步的消息，敬请留意未来几周的公告。


在线文档


随着Mozilla物联网网站的更新，我们开始为WebThings Gateway和WebThings Framework提供一些在线的用户和开发人员文档。你可以通过GitHub（https://github.com/mozilla-iot/docs/），帮助我们完善这些文档。

感谢您为我们精彩的Mozilla物联网社区所作的所有贡献。我们期待这个令人兴奋的新阶段！
