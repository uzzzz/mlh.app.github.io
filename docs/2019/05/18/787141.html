实习生，你真的应该拾起面子，重新上路

  • 柚子社区

那个实习生今天下午收拾东西走人了…

他的离开虽然是意料之中的事情，但是我真的挺伤感的。

他的离开让我有强烈的愧疚感。

我从下午到现在，头脑里始终摆脱不了这件事儿。我有点后悔，为什么他表现不好的时候，不直接训斥他。为什么在他犯错的时候，没有多提醒他几次。

或许这样，他就不会被开了。

下午，他收拾完东西，直接走了，没有跟任何人打招呼。想必他想用这种举动，来表现他内心的愤怒。

他心里肯定是恨我的，我知道。

不然相处了几个月，也教了不少东西给他，至少应该打声招呼再走吧。

其实我很想跟他再聊一次，给他讲讲试用期到底出了什么问题，为什么没有让他留下。

不过在那个节点，愤怒、不满、委屈包裹着着他的内心，我估计说的再多，他也听不进去。

如果他能后知后觉，就能明白我曾经对他说过的话“实习是入行的机会，是你编程生涯中的一个转折点”。

有人引领你成长的机会如此珍贵。稍有脑子的人，都要赶快张开翅膀，借着风尽力飞得高一点。

或许平时你表现不好，被同事、领导说过两句，伤了你的面子。

但面子真的不值钱啊…

真正渴望机会的人，会不惜一切代价去赢取机会。即便是把脸踩在地上又如何？

学到本事，赚到钱就行了啊！！！

实习生，你真的应该拾起面子，重新上路。

其实他实习期的问题，我早就记录下来了。我自认为我还是比较负责的…

可是我一直在犹豫要不要发给他，因为我写的挺直接的…

以后没有机会再提醒他了。索性放出来提醒一下我的学生读者，希望你们以后能少走弯路。

1、在分配任务时，就要搞懂任务是什么。不要自己猜，不要不好意思问。有很多次，你根本不明白任务是怎么，就频频点头，然后还碍于面子，不肯来问。最后，工作没完成，给几个同事都留下了不好的印象。

2、工作就是工作，拿钱办事。不要夹杂自己的情绪，不要觉得任务难就不愿意接。其实你那点心理活动，我们大家都能读懂。遇到一点难度就想放弃，大家都不喜欢这样的伙伴。

3、任务尽量一次做好，不要想着随便写写，出了 Bug 再改。测试同事可不会因为你是实习生，就不给你上报 Bug 。你的 Bug 率直接影响了小组的评分，所以组内的同事才会有所不满。

4、一定要学会主动。主动去学习，任务完成了还要主动的去继续索要任务。任务没完成，就别在公司打游戏了。回家再玩，行吗？你是新人，不要有老油条的做派。

5、工作也是有优先级的，自己小组的工作先做，有时间再给别的组帮忙。职场中，要学会被人拒绝，也要学会拒绝别人。

6、不要觉得自己不行，有点自信。更何况，你嘴巴上叫苦，却没有付出一点行动。

7、踏实一点，浮躁真的不可取。
