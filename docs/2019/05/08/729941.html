关于 LoRa，你需要知道哪些？

作者 | 马智

本文经授权转自智评物联

责编 | 沭七

LoRa 是什么？


LoRa 全名“Long Rang”，是工作于非授权频段的低功耗广域物联网技术，生于 2013 年，单亲，有个来自美国的妈妈——Semtech 公司。LoRa 现已长大，成为在低功耗广域网（LPWAN）或窄带物联网领域里最热门的技术之一。

低功耗广域网 LPWAN 技术的特点是“广覆盖、大连接、低功耗、低速率”，特别适用于远程抄表、环境监测、智能停车、位置追踪等有海量连接、小数据、非频发传输需求的应用场景，并且，这类应用场景的物联网连接数量将占到物联网总连接数的 60%。

看到这里，也许有读者会想起 NB-IoT。Yes，NB-IoT 于 2016 年出生，NB-IoT 全名“Narrow Band Internet of Things”，工作于授权频段的低功耗广域物联网技术。因此，LoRa 与 NB-IoT 同属于低功耗广域网（LPWAN）技术，只不过 NB-IoT 是含着金汤匙出生自全球标准组织 3GPP 这个贵族家庭。从全球范围来看，NB-IoT 目前貌似是 LoRa 相爱相杀的冤家对头。

根据 LoRa 联盟（LoRa Alliance）官网的展示，目前，全球有 100 家网络运营商在 100 个国家已部署 LoRa 网络。那么出生不如 NB-IoT 的 LoRa 为什么能够得到诸如 Google、IBM、思科、Orange、阿里、腾讯等大型互联网、电信运营商、设备商的青睐？

其本质原因在于无线通信所用的频谱也是一种稀缺资源，其中国家划分出一小部分频谱资源作为非授权频段，供 LoRa 之类的网络使用，此外划分出另外一大部分频谱资源作为授权频段，供 NB-IoT 之类的网络使用。

So what？

说通俗一点，任何个人或组织都可以买台 LoRa 基站（也称为 LoRa 网关）搭建 LoRa 网络，好比谁都可以买台 TP-Link 无线路由器搁在家里或办公室用来上网，并且其技术门槛比 Wi-Fi 高不到哪里去。但如果你弄台 NB-IoT 基站搞事情，可被定为扰乱无线电通讯管理秩序罪与破坏公用电信设施罪；也就是说 NB-IoT 这类通信网络只能由政府授权的极少数电信运营商建设和运营，天然带有某种垄断性。


LoRaWAN 又是什么？


话说 LoRa 的美国单亲妈妈——Semtech 公司，当年含辛茹苦一手把她带大。为了使 LoRa 长大成人、受到良好的教育，以便日后出人头地，在 LoRa 两岁的时候，Semtech 联合了 IBM、Actility 等公司成立了 LoRa 联盟（LoRa Alliance），致力于寻找“养父母”一起构建适合 LoRa 发展的产业生态。

LoRa 仅是一种调制技术（NB-IoT 属于蜂窝通信，LoRa 属于线性扩频），属于物理层无线传输技术。因此，LoRa 仅定义了无线侧的标准规范，按照通信的行业术语讲，LoRa 只定义了空口协议，LoRa 对应的产品就是收发器（transceiver）芯片。要想实现组网、完成端到端的通信业务，还需要更多的工作要做。为此，LoRa 联盟（LoRa Alliance）制定了 LoRaWAN 标准规范。

注：LoRa 技术由 Semtech 这一家公司拥有，而 LoRaWAN 则由 LoRa 联盟（LoRa Alliance）这个非盈利组织制定。


LoRaWAN 是在 LoRa 物理层传输技术基础之上的以 MAC 层为主的一套协议标准。LoRaWAN 提供了多信道接入、频率切换、自适应速率、信道管理、定时收发，节点接入认证与数据加密、漫游等特性。有了这个标准之后，LoRaWAN 的就变成一种网络技术，可将其用于不同的协议和不同网络架构（如 Mesh 组网、星型组网、点对点组网）。

以 NB-IoT 做个类比，LoRaWAN 标准的上下行调制技术采用 LoRa，NB-IoT 上行和下行分别采用 SC-FDMA 和 OFDMA。

LoRaWAN 对应的产品包括 LoRaWAN 节点，LoRaWAN 网关（Gateway）、LoRaWAN 网络服务器（Network Server）和应用服务器（Application Server）。

使用 LoRaWAN 的产品时，其用户接口有两个：一个是在底层传感器和 LoRaWAN 节点之间有一个数据接口，传感器数据通过这个接口传到网络；另一个是 LoRaWAN 网络服务器（Network Server）和应用服务器（Application Server）之间有一个数据接口将网络的数据传送至应用系统。

由上可见，LoRaWAN 构建的是一个运营商级的大网，覆盖地区和全国的网络，甚至确保不同国家 LoRa 网络的互操作性。经过几年的发展，目前已建立起了较为完整的生态链：LoRa 芯片、模组、传感器、基站或网关、网络服务、应用服务。


LoRa 目前的发展情况


全球已部署了 5000 万的 LoRa 终端节点，而 2018 年 LoRa 芯片的出货量预计有数千万片，中国市场 LoRa 芯片出货量占据全球一半以上，并且在中国，LoRa 基站的部署量也达到了数万个，在北京、深圳、广州、杭州等城市均有城市级的 LoRa 网络覆盖。

由于 LoRaWAN 的开放性、灵活性和自主性，中国已成为 LoRa 产业生态最大的市场，国内 LoRa 芯片出货量几乎占全球的一半，已有上千家公司的业务与 LoRa 相关。其中，中兴通讯、腾讯、阿里巴巴、中国联通、铁塔公司、贵州广电等重量级玩家加入 LoRa 物联网领域，这些重量级企业的入局对于推动 LoRa 产业发展的作用是不言而喻的。与此同时，国内 LoRa 产业的生态圈也更为完善，以克拉科技为主导力量的中国 LoRa 应用联盟（CLAA）其联盟成员已经增加到了 1100 多家。


阿里巴巴的 LoRa 布局——达尔文计划


在 2018 年 9 月 19 日杭州阿里巴巴云栖大会上，阿里云首席智联网科学家丁险峰宣布启动“达尔文计划”，旨在通过一系列的包括平台、芯片和微基站在内的全链路生态服务，交付给企业客户一张自有可控的物联网。在云栖大会户外上空悬停着一架印有“天空物联网 LoRa 站”Logo 的飞艇，其搭载了 LoRa 网关。当它被启动，现场的物联网设备迅速被飞艇上的信号连接。而当关闭时，数据大屏上的几十种数据会瞬间消失。

阿里巴巴关于 LoRa 的布局主要有三个方面：

  • 第一，阿里巴巴获得了 Semtech 的 LoRa 芯片的 IP 授权，这是 Semtech 在全球继 ST（意法半导体）之后第二次授权 LoRa IP，也是首次在国内授权。在此基础上，阿里云 IoT 联合 ASR（翱捷科技）共同发布了超小尺寸、超低功耗的 LoRa 芯片。

  • 第二，阿里云开发了 LinkWAN 核心网管理平台，该平台是对整个 LoRa 的网关和节点设备进行有效管理的核心网平台。通过这一平台，阿里云 IoT 可以提供基站和节点等设备的接入，为使用者提供终端管理界面，用户可以自主搭配 LoRa 网络，针对不同使用场景推出不同的使用方案。

  • 第三，阿里云针对已有的 LoRa 网关设备和节点设备会进行认证，符合阿里云认证标准的网关设备和节点设备可以接入 LinkWAN 核心网管理平台。


为何 LoRa 受互联网公司的欢迎？


LoRa 的这种灵活、开放、自主的特性给产业界带来更多的机会，因为那些被挡在通信朋友圈之外的企业可以参与到 LoRa 网络运营中。

1. 总体来看，物联网的通信技术可分为短距离通信技术和远距离通信技术。远距离通信技术的连接数量将占到物联网总连接数的 60%。除了 LoRa、Sigfox 等工作在非授权频段的 LPWAN 技术外，远距离通信技术基本上属于电信运营商的地盘；互联网公司只能受限于 Wi-Fi、蓝牙、ZigBee 和 802.15.4 这类短距离通信技术。而 LoRa 技术的兴起正在打破这种格局，提供低功耗广域网络连接服务的厂商已不再是传统电信运营商的“特权”，不同的网络规模和业务范围都有多个新的网络“运营商”参与进来，形成多样化的格局。基于低功耗网络的特点，电信运营商一家独大的局面正在发生改变，给物联网市场带来了新的机遇。

2. 从标准角度来讲，移动蜂窝技术已由高通、爱立信、华为等玩家经数十年的产业资源、技术研发和知识产权积累而占据一隅，互联网公司鲜有可乘之机。互联网企业在通信业标准方面的专利积累非常少、产业链影响力较小，面对 NB-IoT、eMTC 以及未来 5G 等物联网标准，很难参与进来，更不用说有重大贡献和掌握话语权了。不过，LoRaWAN 作为一个问世时间较短且相对简单的开放标准，新入局的互联网巨头相对容易参与到标准化工作中。

3. LoRaWAN 的部署方式和商业模式非常灵活。例如，CLAA 有四种主要的业务合作模式：

  • 独立运营商：提供全套解决方案；支持客户建网，并与 CLAA 共享物联网互联互通。

  • 大型合作伙伴：直接参股，CLAA 负责建网，多城市大范围覆盖，享受全网整体受益，CLAA 承担运维费。

  • 中小型客户：直接采购设备，CLAA 协助建网，城市级或区域级覆盖，享受城市级、区域级收益，客户承担运维费。

  • 专业渠道商：直接采购设备，自行微客户建网，协助客户运营，客户承担运维费用。


LoRaWAN 网络架构


从 LoRaWAN 网络架构来看，主要有以下几部分：终端节点、网关、回传、网络服务器、应用服务器。

  • 终端节点（End Nodes）：终端节点是在最左边，异步地广播数据包到网络。遵循 Aloha 网络规范，保证终端设备可以大部分时间处于空闲模式，功耗少于 1uA。 这种方法可确保在小型电池上的应用可以实现 10 至 15 年的使用寿命。

  • 网关（Concentrator/Gateway）：通过终端节点广播的数据包将会被网络中的一个或多个网关获取。网关有一个多信道和多数据速率的射频嵌入式设备，可以扫描和检测任何活动信道上的数据包并进行解调。

  • 回程（Backhual）：网关通常有一个以太网的回程，然而，目前一些部署使用了 2G、3G、4G、Wi-Fi、NB-IoT、以太网等作为回程。

  • 网络服务器（Network Server）：核心网络是 LoRaWAN 系统的重要组成部分。它承载所有需要的智能来管理网络并将数据分发到其他服务器。

  • 应用服务器（Application Server）：LoRaWAN 协议支持不同类型的网络。一些网络提供商也是应用提供商。因此，在 LoRaWAN 网络架构图图中最右侧的应用服务器可以与网络服务器分别托管或与网络服务器集成在一起托管。


LoRa 的应用场景


LoRa 正在全球范围内应用部署，因为其低功耗、深度覆盖、容易部署等优势使得其非常适用于要求功耗低、距离远、大量连接以及定位跟踪等的物联网应用，如智能抄表、智能停车、车辆追踪、宠物跟踪、智慧农业、智慧工业、智慧城市、智慧社区等应用和领域。
