AWS 首次发布物联网设计集锦—— IoT Atlas！

  • 柚子社区

从 2015 年推出 AWS IoT 以来，通过与客户交流，我们认识到必须对来自传感器网络、控制系统架构以及机器对机器网络等成熟领域的传统物联网设计模式进行变革，才能纳入云开发的概念。

IoT Atlas 是一个方便易用的可搜索网站 （https://iotatlas.net），汇集了各种可用的物联网设计。它假设构建物联网解决方案的人有超大规模的云服务可用，更新并扩展了行业的设计。IoT Atlas 汇聚了各种资源，不论是新入行者还是资深的解决方案构建者都可以找到所需的资源。


由于采用了云服务中性的设计，IoT Atlas 以 GitHub 上的存储库的方式发布。我们依据“知识共享”许可证发布内容，希望能够促进客户、专业领域和 APN 合作伙伴贡献他们的设计理念、注意事项和示例。此外，我们也希望能够激发对解决方案中的新物联网模式的对话和理解。

命令

请求实体可靠地请求某个设备执行单项操作，并确认状态。

设备引导

未注册的设备在某个物联网解决方案中完成注册并全面发挥功能。

设备状态副本

物理设备报告状态或理想未来状态的逻辑表现。

网关

设备在本地设备之间以及设备与云之间扮演中介角色。

软件更新

设备取得新软件，自行执行更新，并确认更新完成。

遥测

从远程环境中的传感器采集数据，并提供测量值以供物联网解决方案的其他组件使用。

遥测档案

保存设备的测量值并以原始或处理后的形式提供使用。

例如，传统以遥测为导向的解决方案可能会从现场设备采集传感器数据，然后使用 FTP、HTTP 或其他机制将该数据发送到某个自我管理的服务器。IoT Atlas 中的遥测设计采用基于云的托管式、可扩展物联网消息收发协议和协议终端节点。这一修改包含了全球性的超大规模云服务，因此令原遥测设计受益颇多。



由于这是我们首次发布物联网设计，我们真诚希望能够通过大家的帮助，让 IoT Atlas 成为一个内容丰富并不断壮大的资源！
