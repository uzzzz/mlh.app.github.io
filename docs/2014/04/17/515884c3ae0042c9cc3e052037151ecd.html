CloudFoundry部署之CF应用部署

一、hello官方测试应用部署

  1. 安装cf命令行，vmc命令行已经被cf命令行所取代
    sudo gem install rb-readline
sudo gem install cf
  2. 确保你的dns设置正确要对应上篇文档中的cloudfoundry01.yml文件中ha_proxy组件的ip 对应域名是*.acp.local(泛域名)
  3. 指定cf target
    cf target api.acp.local

    出现上述错误证明dns解析没有配置好，重新检查dns服务器域名解析设置后再次运行

  4. 登陆target，并且创建阻止和空间
    cf login admin
密码c1oudc0w或者是admin记不太清楚了
cf create-org zhaozhenyang
cf create-space workspace
登陆到自己的space
cf switch-space workspace

  5. 本地创建workspace目录
    sudo mkdir -p ~/workspace
sudo chmod 777 -R ~/workspace
cd ~/workspace
  6. 下载官方测试代码
    git clone https://github.com/Altoros/cf-vagrant-installer.git

  7. 进入下载后的目录
    cd cf-vagrant-installer/test/fixtures/apps/sinatra
  8. 修改应用的清单文件(manifest.yml)
    vi manifest.yml
修改domian的值为acp.local

  9. 上传应用
    cf push

  10. 最后通过浏览器访问也可以通过curl访问
    浏览器：https://hello.acp.local
curl hello.cf.local
